多分辨率训练 + CLTagger V2 + 自定义底模 + 模型根目录可迁移 + 设置页重做 + 错误提示本地化 + 自更新/ONNX 多项修复

### 新增

- 多分辨率训练：一次混训多个基准分辨率 + 桶长宽比可配 + 桶分布预览（#296）
  官方示范 LoRA 常用多个分辨率（如 512/768/1024）混训改善高分推理；此前炉子只支持单一基准分辨率，大长宽比的作品还会被写死的 2:1 桶强裁。本版在保留原有按长宽比分桶的前提下加入多分辨率训练。

  - 训练分辨率可填多个值（如 `512, 768, 1024`），无前缀文件夹的每张图在各分辨率各训一遍。
  - 文件夹名加 `1024px_` 前缀（如 `1024px_2_data`）可固定该文件夹分辨率、覆盖全局列表。
  - 新增 `aspect_ratio_limit`（默认 2.0），调大让长图按原比例进桶、不再被强裁。
  - 训练页右栏显示实际桶分布（每桶尺寸 + 有效图数），样本凑不够一个 batch 的桶会标红提醒（整桶会被丢弃）。
  - 顺手修：基准分辨率设为 512 时原来只剩一个方桶（分桶失效、所有图被压成正方），现已按基准分辨率正确生成桶范围。
- 新增 CLTagger V2 打标器（gated 仓库 + 外置权重，V1 行为不变）（#297）
  打标器 CLTagger 新增 V2 版本（`cella110n/cl_tagger_v2`）支持。V2 的权重和词表结构与 V1 不同（2GB 权重是外置文件、独立词表、RGB 预处理）。

  - V2 是 gated 仓库，下载时自动带上你在密钥页填的 HuggingFace token；缺 token 或未授权时给出可操作的提示。
  - 外置权重文件纳入完整性校验，避免只下到一半就被误判「可用」、到加载时才黑盒报错。
  - V1（`cl_tagger_1_02`）行为完全不变，可继续使用；在 Settings 打标区切换版本。
- 可注册本地微调权重为底模 + 正则先验/测试出图临时换底模（#302, #309）
  此前底模只能从官方提供的几个版本里选，想在自己的微调权重上测试出图、或拿本地下载的 `.safetensors` 当底模没有第一方入口。本版补上。

  - Settings → 训练 → Anima 主模型可「添加本地模型」，把盘上已有的 `.safetensors` 注册成底模（只记路径，不复制、也不删原文件）。
  - 正则集 AI 先验出图、测试页出图各加一个「底模」下拉，可临时换别的已下载版本（含本地微调权重）来生成，不动全局设置；下拉只列磁盘上确实存在的模型。
  - 顺手修：测试/出图页此前其实没跟随 Settings 里选的底模版本（一直写死用一个版本），现在真正跟随，在微调权重上测试才生效。
- 模型根目录可改到其它盘 + 一键迁移已下载权重（无需重启）（#304）
  下载的模型权重默认落在仓库下的 `models/`，越下越大。和 studio_data 数据目录一样，现在可以把模型根目录改到其它盘，并一键把已下载的权重复制过去。

  - Settings → 系统 → 存储位置新增「模型根目录」卡；更改位置 → 扫描确认（文件数 / 总大小）→ 进度条复制。
  - 复制后即刻生效、无需重启（这点与 studio_data 迁移不同）；旧目录原样保留，旧训练配置里写死的旧绝对路径仍能解析。
  - 有训练 / 打标等任务在跑时迁移会被拒绝，避免搬动过程中文件被截到写一半。

### 变更

- Settings 加密钥 tab 集中凭证 + 下载源改为按模型类型独立选（#301）
  设置页的凭证和下载源做了重组。

  - 新增「密钥」tab 集中 HuggingFace / ModelScope / Gelbooru / Danbooru 凭证（此前散在训练 / 数据集 / 监控等多个 tab，比如 CLTagger 要用的 HuggingFace token 得去训练页找）。
  - 下载源从「一个全局 HF/ModelScope 开关」改成各模型卡按类型独立选（训练模型 / WD14 / 放大器各一个下拉）；老的全局设置会作为初始值自动迁移，不静默回退。
  - 下载失败时的错误提示改得更可操作（如 gated 仓库缺 token 直接提示去填 token），不再只在终端可见。
  - 原本挂在 Gelbooru 下、实际被所有下载 / 上传共用的几个字段（保留 tag、转 PNG、去 alpha 通道）搬到全局下载设置下，旧值自动迁移不丢。

### 改进

- 错误提示按 app 语言本地化（中/英），ADR 0009 错误信封迁移完成（#294）
  此前 app 虽是中英双语，但出错时的 toast 文案写死中文，英文用户看到的是中文错误信息。本版起错误提示按 app 语言本地化。

  - 覆盖约 120 个错误码，中英双语；下载失败等场景的错误信息也更具体、可操作（如 gated 仓库缺 token 会直接提示去填 token）。
  - 后端每个错误响应统一带语义错误码 + trace ID，前端按错误码查本地化文案；暂时还没覆盖到的错误码会回退显示原始英文信息，不会变空白。
  - 至此 ADR 0009 错误信封的三阶段迁移（统一信封 → 错误码本地化 → 清理旧字段）全部完成。
- Settings 页字段标题/布局/开关样式统一整理 + 移除冗余选项（#306）
  设置页做了一轮 UI 整理，统一长期积累的不一致。

  - 字段标题改成人话（不再直接用配置字段名，如 `exclude_tags` 显示「排除 Tags」、`api_key` 显示「API Key」）。
  - 布尔开关统一改成启用/关闭下拉（修上下对齐），表单控件宽度和样式统一，内容区最大宽度放宽到适配宽屏。
  - 系统页「重新运行首次引导」并入「服务」区；存储 / 模型根目录地址从代码块改成可选中复制的只读输入框。
  - 打标器手动指定本地模型目录的选项移除（模型根目录已能统一指定落点，该选项冗余）；旧配置里的残留值加载时自动丢弃。
- 训练监控采样图角标显示 epoch+step + 移除失效的独立监控按钮（#307）
  - 训练监控页的采样缩略图 / 大图角标现在显示 epoch + step（此前 epoch 只藏在文件名里，要点开放大看文件名才知道是第几轮）。
  - 缩略图角标从叠在 64px 小图上（盖住画面）改到图下方一行紧凑显示，图本身保持干净。
  - 移除一直失效的「独立监控」弹出按钮（漏了 app 路径前缀、点击会落到错误页）；侧边栏「监控」入口本就带任务选择，这个按钮是冗余的。

### 修复

- 修复 Linux 装 onnxruntime-gpu 误装 CUDA 13 build 致打标崩溃（#311）
  Linux 用户装 ONNX CUDA 版后，打标直接报 `libcudart.so.13` 找不到并误报「未安装 onnxruntime」——其实包装上了，只是加载失败。

  - 根因：新版 onnxruntime-gpu（1.27 起）的 PyPI 默认 wheel 切到了 CUDA 13，而本项目整套按 CUDA 12 设计（torch 是 cu128、预加载的库都是 `.so.12`），混用导致加载期就崩。
  - 本版按 torch 的 CUDA 大版本分流安装对应的 onnxruntime-gpu build，CUDA 12 钉到 1.20–1.26 这条线；将来 torch 升到 CUDA 13 时会自动切到对应 build。
  - Settings → ONNX Runtime 信息盒加一行显示当前 torch 的 CUDA 大版本，错位时带提示。
  - 已经装坏的用户可手动 `pip install "onnxruntime-gpu==1.20.1"` 后重启立即恢复；DirectML / CPU / Windows 路径不受影响。
- 自更新加固：npm 改写 lockfile 不再阻断升级 + 老用户升级不重下模型数据（#308, #310）
  WebUI 自更新（`git fetch` + `git reset --hard`）的两类可靠性问题修复。

  - 新版 npm 在启动期会自动给 `package-lock.json` 追加 `peer` 等元数据，让这个受版本控制的文件凭空变「脏」，把「工作树必须干净」的更新前置检查卡死、更新按钮点不动。本版起这类自动改写不再算脏、不阻断更新。
  - 若工作树真有手动改动，更新前会弹「强制覆盖并更新」确认（回滚也共用），避免 reset 静默覆盖用户的真实改动。
  - 早期版本曾把几个下载得到的模型数据文件（tokenizer / 配置，十几 MB）误入 git 库、后来转成 gitignore；从那个时期的老版本升级时 `reset --hard` 会把它们删掉、逼用户重下。本版在升级前备份、升级后还原这几个文件，老用户零感知（大权重文件从未入过库，不受影响）。
- 修复小数据集 grad_accum 尾组梯度被丢（每轮少训一步）（#305）
  数据集每轮的 batch 数不能被梯度累积步数（`grad_accum`）整除时，最后不满一组的那几个 batch 的梯度会被丢弃、不计入训练。

  - 极端情况：3 张图 / `grad_accum=4` 时整轮一个 update step 都没有、完全不训练。
  - 多轮训练时尾组梯度还会泄漏进下一轮、并被错误归一。
  - 本版起每轮末尾不满一组也照常 step，并按实际 batch 数归一（末步梯度不被削弱）。已能整除、或 `grad_accum=1` 的配置行为不变。
- 修复关缓存 + batch>1 + 非方图组合训练 torch.stack 崩溃（#299）
  关闭 latent 缓存（`cache_latents=False`）+ batch 大于 1 + 训练集长宽比不全相同时，训练会直接报 `stack expects each tensor to be equal size` 崩溃。

  - 该路径此前漏了按长宽比分桶，一个 batch 里混入不同尺寸的图，拼接时崩。
  - 缓存路径（默认开启）早已规避；本版让非缓存路径也按桶分批。多分辨率训练（本版新增）关缓存时尤其容易触发，已一并修好。

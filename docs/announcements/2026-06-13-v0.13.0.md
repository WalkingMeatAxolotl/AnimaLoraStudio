采样链路统一 Comfy parity + T-LoRA / OrthoLoRA 适配器 + 失败 / 取消任务可继续训练 + 测试页历史回填 + 首次启动引导 + 项目页搜索 / 归档 + studio_data 自定义位置

### 新增

- 测试 / 训练采样 / RegAI 三条统一 Comfy parity runtime v2（#256）
  测试页出图、训练采样预览、RegAI 先验生成三条采样路径合并成同一套实现，全部对齐 ComfyUI KSampler 行为；老的 parity 开关移除。

  - 采样器新增 `dpmpp_3m_sde`、调度器新增 `sgm_uniform`，与 ComfyUI 同名版本逐参数对齐。
  - 训练采样新增「Comfy 编码」开关（默认开）：caption 原文整段送进 Qwen3 + T5，booru 括号写法保持字面；关闭走旧版逐 tag 编码，方便新旧 A/B 与按旧编码断点续训。
  - 负面提示词对齐 ComfyUI：留空即空串，不再隐式塞默认负面 prompt。旧预设里若 sampler / scheduler 是已不支持的值（如 `euler`），自动归并到默认值。
  - Settings → 测试 新增 VAE 解码精度下拉（bf16 / fp32），默认 bf16；现代显卡下默认路径不再做不必要的模型搬运。
  - 出图过程中偶发的注意力 NaN 退到 fallback 实现，跑完当前这张图后自动复位，不再永久降级整个进程。
- T-LoRA + OrthoLoRA 适配器 + Automagic v2 实验路径（#215, #258）
  训练配置 `lora_type` 新增两个变体，定位小数据集人物 / 主体定制。

  - **T-LoRA**（arxiv 2507.05964）：按 timestep 自适应 rank，高噪声段限制 rank、低噪声段满 rank，缓解小数据过拟合。
  - **OrthoLoRA**：在 LoRA 基上叠一层正交旋转 + 缩放，可单独选 `ortho`、也可与 T-LoRA 叠加（开关 `tlora_use_ortho`，默认开 — T-LoRA 论文的完整配方就是 ortho 变体）。**保存时自动蒸馏成标准 LoRA**，ComfyUI 等推理端零成本加载。
  - `lora_type` 各选项的描述全量重写为「参数干嘛 + 适用场景」，不再只罗列算法名词。
  - 实验性 Automagic v2（per-param scalar lr + fused backward 省显存）：UI 不暴露，需手改 `studio_data/secrets.json` 把 `enable_automagic_v2` 设为 true 才出现。CLI / yaml 路径不受影响。
- 失败 / 取消的训练任务可从 epoch 备份点继续训练（#266）
  上一版起每个 epoch 末尾就在写自动备份点，但「继续训练」按钮此前只对 paused 状态开放 —— failed（崩溃 / 关机）和 canceled 任务的备份点其实多数完好。本版打开。

  - Queue 列表 / Queue 详情页对 failed / canceled 行加「继续训练」按钮，与原有「重试（从头）」并列。done 状态仍只能重试。
  - 关机重启后照样能恢复：备份点位置在数据库里有记录，进程或机器一死也不丢。0.12 时代的存量任务会自动回填到新机制下。
  - 自动备份点改落任务档案目录（`studio_data/tasks/<id>/state/`），与采样图、运行日志同处，便于整体归档。用户自己周期性 save 的训练 state 位置不变。
  - 写盘改原子方式，断电也不会留半截备份文件；恢复成功后不再删除备份点，避免「刚 resume 到下个 epoch 之间」中间没有恢复点。
- 测试页历史点击回填参数 + 落盘 PNG 自带参数快照（#239, #244, #245）
  测试页历史栏从「只切大图」升级为「连同参数一起恢复」；新增 Settings → 测试 → 自动落盘开关，开启后图片直接存到 `studio_data/test/<日期>/`。

  - 历史栏点击：xy 模式同时回填 xy 轴 + LoRA 列表，single 模式回填 LoRA + 张数。误点给 5 秒撤销提示。
  - 落盘的 PNG 内嵌完整参数快照，拖回 Studio 可读、拖去 ComfyUI / WebUI 也能解出 prompt（single 模式同时写入 a1111 兼容格式）。
  - 跨会话 / 清缓存 / 换设备：Studio 启动时扫描落盘目录补齐历史栏，不再依赖浏览器本地存储。
  - 历史栏每张图左上角的 × 按钮拿掉（误触率高），单条删除走顶部「清空」/「清失效」。
- 首次启动引导 modal — 基础能力 checklist + 一键安装（#238）
  新人第一次开 app 不再需要在训练 / 打标时撞墙再回 Settings。语言 modal 选完语言后立刻弹出引导。

  - 4 条 checklist：底模套件（Anima 主模型 + VAE + Qwen3 + T5）、打标功能（WD14 + ONNX Runtime）、训练加速（flash-attn / xformers）、图像增强。任一未装显示「未安装 (X/N)」，勾选 → 一键开始下载。
  - 中文默认 ModelScope、英文默认 HuggingFace，自动写入下载源。装中实时显示下载速度 / ETA / 当前文件。
  - Settings → 系统 加「重新运行首次引导」按钮，可随时调出。
- studio_data 目录可自定义位置 + 复制迁移（#263, #267）
  装在系统盘的用户可以把不断增长的项目数据挪到大容量盘。Settings → 系统 → 存储位置，选目标目录后弹出确认 modal（显示待复制文件数 / 总大小 / 顶层明细），开始迁移后进度条实时推进。

  - **只复制不删除**：旧位置数据原样保留；迁移失败时清掉半截目标目录、当作没发生过。
  - 数据库在线快照备份，避免迁移期间被截到写一半的状态。
  - 在所选目录下会创建 `studio_data/` 子目录搬入（不会把内容散到你选的父目录里）。
  - 若指针文件损坏 / 盘符未挂载，自动回退默认位置，旧数据不丢。
  - 顺手修：新创建的 version 目录不再生成空的 `samples/` 文件夹（采样图实际落在任务档案目录）。
- 项目页加搜索 / 过滤 / 排序 + 软删除归档 + 「准备中」显示当前阶段（#265）
  项目多了之后的浏览体验改造：找得到项目、不活跃项目能收起、能看出每个项目卡在哪一步。

  - header 「过滤 ▼」开关折叠时不占空间；展开后是一行：搜索（标题 / slug / 备注，大小写不敏感）+ 状态下拉 + 排序下拉。
  - 卡片右上角 × 从硬删改**归档**：目录 / version / 任务全保留，可随时恢复。header「已归档（n）」可切换到归档视图，归档卡片半透明，提供恢复和真删两个按钮。归档不更新「最近活跃时间」，恢复后项目回到原位置不会顶到最前。
  - 状态徽章「准备中」改为显示当前阶段（如「准备中 · 打标」），6 个阶段全覆盖（下载 / 筛选 / 预处理 / 打标 / 正则集 / 训练）。
- 测试出图模型闲置自动卸载释放 VRAM（#232）
  - 解决「切到别的页面忘了清理，下次开训练就爆显存」的常见场景：测试页加载完模型后空闲超过设定时间，自动卸载释放显存。
  - 默认 10 分钟；Settings → 测试 可调，设 0 关闭。运行中改值即时生效。
- 打标器加「已有 caption」策略：覆盖 / 跳过 / 追加（#237）
  - 打标页顶栏「格式」旁新增「已有 caption」下拉：**覆盖**（默认，保留原行为）/ **跳过**（已有 caption 的图直接保留）/ **追加到末尾**（按 tag 合并去重写回原格式）。
  - WD14 / CLTagger / LLM 三个打标器都支持。追加模式下 trigger word 不会被重复加。
  - 本次任务专属选项，不进全局 Settings。

### 变更

- 全 app 任务日志统一为三态抽屉（下载 / 预处理 / 打标 / 正则集 共 6 页）（#253）
  各页面的任务日志原本五套不同实现、行为各异，本版统一成挂在底部的三态抽屉。

  - 无任务时整体隐藏；有任务时默认显示一行收起条（状态徽标 + 最后一行日志 + 耗时），不展开也能看到进度在动。
  - 进入运行状态自动展开（含回到页面发现任务还在跑的回放场景）；任务结束不自动收起，需要手动收或切页。
  - 顺手修：正则集页的历史 booru 任务不再永久遮住正在跑的 AI 先验日志；预处理的放大 / 裁剪任务获得日志回放；去重扫描失败的错误进日志（此前只有 toast）。
- ONNX Runtime 改 Settings 手动装 + 新增 DirectML 支持（#236）
  - Studio 启动期不再自动 pip install onnxruntime（~150MB / 几分钟），未装时静默跳过；启动更快，不用 WD14 / CLTagger 的用户不被阻塞。
  - 新增 DirectML 后端：走 DX12，**不依赖 CUDA / cuDNN / cuBLAS**，任何 DX12 GPU（NVIDIA / AMD / Intel，Windows-only）都能跑。修复 RTX 5090 上原 onnxruntime-gpu 静默降级 CPU 的问题（issue #231）。
  - Settings → ONNX Runtime 三按钮：DirectML / CUDA / CPU；自动检测按平台分流（Windows + GPU → DirectML / Linux + GPU → CUDA / 无 GPU → CPU）。
  - 打标页未装 onnx 时显示「去安装 ONNX Runtime →」按钮，一键直跳 Settings 对应区域。
- Tag 翻译词典默认源换成 ffdkj（每日更新，31 万条）（#268）
  - 原默认源 Physton 已停更 2.5 年（2023-11 最后更新，9.9 万条）。换成 ffdkj，每日更新、覆盖 31.1 万条（post_count ≥ 10 的全部 tag，Gemini 翻译 + 人工校对）。
  - 按热度取前 20 万条加载，截掉的是引用数最低的长尾。
  - 用户上传自定义词典的流程不变；Settings 「恢复默认词典」按新源重新下载。

### 修复

- LoKr 分层 rank 推理强度与训练对齐（#241）
  分层 rank 训练的 LoKr 之前在 ComfyUI 等推理端加载时强度与训练不一致。本版保存的 metadata 会写出每层实际的 rank 与 alpha，推理端按层读取做缩放，与训练前向路径一致。
- Booru 下载区分 429 / 503 + 不再永久减速 + 日志去重（#234）
  - gelbooru 服务端偶发 503 时，原日志被误写成「429 backoff，等 59.5s」误导排查方向，且单次 503 会把下载速率永久减半（不可恢复）。
  - 区分两种语义：429（「你太快了」）→ 退避 60 秒 + 减半速率；503（「我挂了」）→ 退避 15 秒，**不动速率**。
  - API 和 CDN 各自独立退避，CDN 风暴期间 API 搜索不受连累。同一退避窗口内连续 503 只打 1 行日志，不再刷屏。
- Windows 部分机器开 Studio 白屏（主动注册 .js / .css MIME 类型）（#233）
  部分 Windows 机器因为系统注册表被其它软件改过，浏览器拿到 `.js` / `.css` 的 MIME 类型不正确，导致 Studio 加载白屏。后端主动注册正确类型回避注册表读取。
- 标签 chip 模式输入框 Backspace 不再误删已确认的标签（#247）
- 预处理质量候选审核面板布局 — 卡片响应式多列 + 高度占满（#252）
- UI 小修：日志回放 / 打标 skip 前置过滤 / 裁剪残留 / 去重面板（#249）
  - 切页面 / 刷新后恢复 RegAI 先验生成页和打标页的实时日志。
  - 打标选「已有 caption = 跳过」时，提前过滤掉已有 caption 的图，避免空转加载打标器。
  - 训练范围裁剪完成后删除不再属于输出的旧源图及 sidecar，修 bundle / 版本复制后原图 + 裁剪图同时残留的问题。
  - 去重质量审核面板改切换显示模式（分组 / 质量），防窄屏面板互挤；打标高级选项展开不再挤掉日志面板。
  - Bundle 导入导出保留 `version_label` 与 `preset_name`，导入时还原而不是固定为 v1。
  - 复选框 / radio 颜色统一为主题强调色。

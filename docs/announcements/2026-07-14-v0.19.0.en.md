Retouch & mask brushes with masked loss · zoom and pan everywhere · latent cache in bundles · version-switch consistency fixes · clean Ctrl+C shutdown

### Added

- **Retouch tool: paint over text and watermarks (#393)**
  Preprocessing gains a fifth tool, Retouch: paint over text, watermarks, and clutter with a solid color-picked brush so your LoRA never learns them. Alt-click to sample a color, soft-edged brush, per-image undo/redo (Ctrl+Z); edits stay in memory until you save one image or all at once.
- **Mask brush + masked loss: training skips the regions you mark (#394, #395, #400)**
  The Retouch page gains a Mask mode: brush over regions you don't want trained on (hands prone to artifacts, irrelevant backgrounds), with an eraser and a coverage readout (warns above 50%). Then enable `masked_loss` in the training config's loss group — masked regions are excluded from learning, so generated images fall back to the base model there instead of picking up artifacts from your dataset; grayscale masks give partial weighting. Masks are stored next to each image as `{name}.mask`, follow crop and upscale automatically (restoring the original deletes them), and can optionally be included in bundle exports.

  - `masked_loss` is off by default: the training page points it out when your set has masks but the switch is off, and mask changes automatically invalidate the latent cache. Not yet compatible with NaViT packing (masks are ignored there with a log notice).
- **Bundle export can include the VAE latent cache (#391)**
  When exporting a bundle.zip, you can now include the VAE latent cache (`.npz`) for the training set and regularization set separately; importing on another machine reuses it directly and skips re-encoding (biggest win on machines where encoding is slow). Expect a noticeably larger bundle (about 1 MB per 1024² image, doubled with flip augmentation).
- **Tagging page shows validation set progress (#392)**
  When a version has validation images, the tagging page's right panel gains a "validation set" section: tagging progress and the "to tag this round" estimate are tracked per dataset, and selecting the validation scope no longer shows "—" for the training side.

### Improved

- **Zoom and pan in every single-image view (#396)**
  All single-image views (fullscreen viewer on the Generate page, gallery preview modals across the app, the inline image on the tagging page) now support pointer-centered wheel zoom, drag to pan, and double-click to toggle fit / 100%; the tagging page image is upgraded to full resolution, so zooming in stays sharp.

  - Clicking the image inside a viewer no longer closes it (click and drag-to-pan can't coexist); close with the × button, Esc, or by clicking the edge overlay.
- **Tag autocomplete: suggestions only pop up as you type + global toggle (#401)**
  Clicking into an input to move the cursor no longer pops suggestions over what you're reading — they appear only after the text changes, and clicking the input dismisses an open list. Settings → Tag dictionary gains an "autocomplete while typing" switch to turn it off app-wide.
- **Failed evaluations can be retried from the log entry (#397)**
  When post-training evaluation fails, the evaluation log entry now has a Retry button that re-runs the whole round in one click; previously you had to re-select checkpoints manually. The "Retry" button in the task detail header is renamed "Retrain" for training tasks, making clear it starts training over with the same config rather than re-running evaluation.
- **Readable errors when the T5 tokenizer is missing (#387)**
  When the local T5 tokenizer directory is missing, training downloads it automatically; on flaky networks this used to fail with an inscrutable timeout stack trace. The download is now announced in the log, and failures explain the cause with fix guidance (check your network / download it from Settings / verify the path setting).
- **Curation dataset switch matches the queue page style (#392)**
  The training / validation set switch changes from two tabs to the same single-button toggle as the queue page (labeled with the set you'd switch to), placed at the far right of the header.

### Fixed

- **Stale data after switching versions (#389, #390)**
  In multi-version projects, after switching versions: clicking "Start training" right away could enqueue the previous version, and step pages could keep showing the previous version's data — the curation page in particular kept showing old images while actions applied to the new version. Switching now takes effect immediately and step pages fully reload with the version; switching away from unsaved tag edits asks for confirmation first.
- **Ctrl+C in the terminal couldn't stop the server (#398)**
  With a browser tab open, pressing Ctrl+C used to hang the server at "Waiting for connections to close" forever (a second Ctrl+C didn't help; only killing the process did). One Ctrl+C now exits cleanly within about 3 seconds, without the spurious errors that used to appear during shutdown.
- **Windows training logs no longer show a misleading triton error (#388)**
  With xformers enabled on Windows, logs used to print a harmless `No module named 'triton'` traceback — and when a task failed for an unrelated reason, it could be shown as the failure cause, suggesting you needed to install triton. The probe is now skipped (triton plays no part in the xformers kernels this app uses), keeping logs and failure summaries clean.

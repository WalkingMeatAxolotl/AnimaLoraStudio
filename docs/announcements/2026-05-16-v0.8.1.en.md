Single-view version panel + update channel decoupled from git (ADR 0005) + one-click self-update for zip users

### Added

- **One-click git-repo initialization for zip-install users (enables self-update)**
  ADR 0002 only supported git-clone deployments; because zip-extract users have no `.git/`, the version panel showed everything as unknown and self-update didn't work at all. This release adds automatic normalization:

  - **Banner at the top of the version panel**: when zip mode is detected, it shows an "Enable self-update" button with explanatory text; if git isn't installed, the text changes to "Install git first" with a link to the official site
  - **Bootstrap flow**: `git init` → `git remote add origin <URL>` → `git fetch origin master --tags`, anchoring on the `v{__version__}` tag when possible (so the working tree looks clean); the reset is `--mixed`, leaving the working tree untouched and your original zip files in place
  - **Configurable upstream URL**: fork maintainers can override the default via the `ANIMA_STUDIO_ORIGIN_URL` env var
  - The "dev-branch zip" case isn't supported (end users run the master zip; dev zips are a developer self-testing scenario)

### Changed

- **Version panel switched to a single view + the upgrade channel is now a user preference rather than git state (ADR 0005) (#81)**
  ADR 0005: the "channel" used to be tied to git working-tree state (branch name + `git reset --hard`), so right after a release you'd get the contradictory reading of "v0.8.0 installed" + "↑2 commits behind" + "switching to dev does nothing". The new model treats the channel as a **user view preference**:

  - **Only the current channel is shown on screen** (no more master and dev side by side), avoiding the visual confusion of "both channels are live"
  - **Toggling doesn't touch git** — it just writes `system.update_channel` to `secrets.json`; the real "switch to dev HEAD" / "update to vX.Y.Z" are separate buttons
  - **A new backend "what's installed" classification**, `installed_kind` (stable / dev / custom), inferred by comparing commit hash + tree, replacing the frontend's reliance on reading `branch`
  - **Wording free of git jargon**: "↑N commits behind" → "New stable version vX.Y.Z available" / "Up to date"; commits / sha / branch no longer appear
  - The old `show_dev_channel` is migrated once to `update_channel`, dual-writing on save for backward compatibility

### Fixed

- **Three version-panel fixes: wrong "switch to dev" button state / a phantom arrow for the same version / a duplicated button (#82)**
  Follow-ups to the ADR 0005 rework:

  - Rollback wording now branches on the stable / dev `installed_kind` instead of always showing "Roll back to vX.Y.Z"
  - The preflight panel no longer shows a contradictory switch target when the channel preference and git state disagree
  - dev-card fetch race: `devCommits` and `devCheck` are split into separate useEffects, so one resolving first can't trigger a re-render that skips the other's fetch (symptom: the "Switch to dev HEAD" button looked enabled but was a no-op when clicked)

Studio's URL changed from `http://<host>:<port>/studio/` to the root path `http://<host>:<port>/`.

- Launching via studio.bat / `studio run` opens the new URL automatically — most users won't notice.
- Old `/studio/...` bookmarks still work: they redirect to the new URL, but updating the bookmark to the root URL is recommended.
- Reverse-proxy deployments (e.g. HuggingFace Space, or rented GPU boxes that proxy the container port to the root path) now load correctly.

If the page is blank after updating, it's usually a cached old page: hard-refresh (Ctrl+Shift+R) or open the root URL in an incognito window.

Small UI improvements + a cross-platform onnxruntime-gpu fix

### Improved

- **The tagging filter page gains a fullscreen preview and keyboard accept/remove shortcuts (#27)**
  - A fullscreen preview replaces the popup preview
  - Keyboard accept / remove shortcuts make working through images one at a time faster
  - A clear CNB export entry point after you save tags

### Fixed

- **onnxruntime-gpu silently downgraded to CPU on Windows / Linux (#29, #30)**
  Root cause: when the CUDA EP fails to dlopen, onnxruntime doesn't raise — it silently falls back to CPU internally; `ort.get_available_providers()` still reports CUDA as available, the UI looks fine, and all you see is CPU usage spiking.

  - Added monitoring: `_create_session` compares against the actual `session.get_providers()`, and if CUDA was requested but isn't actually active it reports `cuda_load_error` so the UI can surface it
  - Windows: Python 3.8+ dropped automatic DLL loading from PATH, so we added `os.add_dll_directory(torch/lib)` to let onnxruntime find torch's bundled `cublasLt64_12.dll` / `cudnn_*.dll`
  - Linux: the worker subprocess now explicitly does `import onnxruntime_setup` at the top to trigger preloading; fixed a misjudgment in `_has_system_cuda_libs()` (cloud images with the CUDA Toolkit but no cuDNN used to be mistaken for a complete system CUDA install, which skipped preloading)
  - Added a `tools/diagnose_onnx_gpu.py` diagnostic script

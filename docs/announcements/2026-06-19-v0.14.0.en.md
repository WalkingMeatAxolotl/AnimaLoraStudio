New SOAP optimizer, LeapAlign self-distillation, and SRA v2 alignment training + Generate page redesign + fix for VAE stalls on high-VRAM cards

### Added

- **New SOAP / Schedule-Free SOAP optimizers for faster convergence on matrix-type adapters (#271)**
  - Two new optimizer types: `soap` (Adam in the Shampoo eigenbasis) usually fits the low-rank matrix factors of LoRA / LoKr faster than AdamW / Prodigy, at the cost of more state VRAM.
  - `soap_sf` layers a Schedule-Free trajectory on top of SOAP — no learning-rate schedule needed, and it eases abrupt style shifts. Like PPSF, it requires the scheduler to be set to none.
- **New LeapAlign two-step leap self-distillation training, with four FlowBP trajectory variants (#277, #278)**
  - Turn it on with `leap_enabled` (under the advanced training options). It builds a two-step leap trajectory on rectified flow so the leap-predicted latent approaches the real latent of the training data — no reward model or full sampling trajectory required, and it fits LoRA fine-tuning.
  - `leap_ratio` controls how much it mixes with standard training; `leap_variant` offers four trajectory-integration methods (following the FlowBP approach): original / sparse / bridge / lagrange, with original as the default to keep behavior unchanged.
  - Mutually exclusive with InfoNoise, loss_weighting, and huber loss (grayed out via the linked front-end controls); these mechanisms are skipped when LeapAlign is on.
- **New SRA v2 self-representation alignment auxiliary loss, aligning intermediate representations to the VAE latent (#279)**
  - Turn it on with `sra_enabled` (off by default, under the advanced training options). It projects the intermediate hidden state of one transformer block back into the VAE latent space and aligns it with the clean latent, acting as an auxiliary regularizer that aids convergence without an extra diffusion forward pass.
  - `sra_weight` (default 0.2) controls the weight and supports none / linear / cosine / jump decay schedules; the projection MLP exists only during training and isn't written into the exported LoRA.
  - `sra_normalize` is on by default: before alignment it standardizes the prediction and target per sample, preventing oversized video-VAE latent scales from throwing the alignment loss off balance.
  - Only attaches to the standard rectified-flow path; not used together with LeapAlign.
- **Project cards can now edit their title and notes, and the action icons appear on hover (#286)**
  - Project cards gain a pencil button for editing the title and notes (notes can be used for search / categorization); the slug and LoRA output name stay fixed, so changing the title doesn't affect any output file name or path.
  - The action icons at the bottom of the card (edit / archive / restore / delete) now fade in on mouse hover or keyboard focus, for a cleaner look the rest of the time.
- **Regularization sets gain a global default set of excluded tags, so you don't re-enter them when switching projects or versions (#285)**
  - Settings → Dataset → Regularization adds a "default excluded tags" field; the regularization set generation page uses it as the initial exclusion seed when that build has no local record yet (common meta tags like watermark / signature / username no longer need to be retyped every time).
  - It's only a seed: you can still add or remove individual tags on the page — it's not a hard filter.

### Changed

- **The Generate page's left-hand config moves to paged tabs, and the prompt picker gains thumbnail previews (#280)**
  - The three blocks on the left — LoRA/XY · Prompt · Config — change from a vertical stack to paged tabs inside one unified card, showing one page at a time, with the current page remembered locally.
  - The prompt picker adds a thumbnail per row and a larger preview at the bottom, making it easier to compare against the generation results on the right.
  - The XY-axis card is tidied up: the axis name gets its own row, the remove button moves to the right end instead of sharing a row with the dropdown, and weights accept a single comma-separated entry (e.g. `0.2, 0.3, 0.4`).

### Fixed

- **Fixed occasional VAE stalls of up to hundreds of seconds during sampling / image generation on high-VRAM cards (#281)**
  - On high-VRAM cards (e.g. 32G), when the working set of a full-image VAE encode/decode approaches about half of VRAM, it triggers system VRAM paging — a single operation degrades from under a second to hundreds of seconds, committed system memory spikes by 20-30G, and it doesn't raise a clean CUDA OOM, so the old "only tile after an error" fallback couldn't catch it.
  - Adds a `vae_tiling` setting (auto / on / off, default auto): it predicts based on live available VRAM and proactively tiles the encode/decode when crossing the cliff, while staying on the fast full-image path with no performance penalty when it doesn't.
- **Fixed the LLM tagger prompt-template input losing focus after a single character (#291)**
  - In the LLM tagger's preset prompt template (the message editor), the input lost focus after every character typed, making continuous typing impossible and forcing repeated clicking. This is now fixed and typing works normally.
  - The same editor is shared between the tagging page and Settings; both are fixed.
- **Fixed the DirectML build of onnxruntime permanently reporting that Studio needs a restart (#292)**
  - Users with onnxruntime-directml (the DirectML backend) installed saw the Settings → ONNX Runtime card persistently showing "restart Studio / the current process is still using the old one," which no number of restarts could clear — even though DirectML was actually working fine.
  - The cause: that package's version number simply differs from its internal onnxruntime core version, and the old logic misjudged a restart was needed based on it. This version judges by the actually loaded execution provider (EP) instead, so it no longer reports falsely.
- **Fixed the pause button only appearing after switching pages once training resumed or started (#287)**
  - After resuming (terminal / paused → resume) or just starting training, the pause button didn't appear on the Monitor / Queue / Overview pages — you had to switch to another page and back for it to refresh.
  - All three pages now listen for events such as the first epoch backup being written to disk and refresh the button state in real time, with no page switch needed.
- **Fixed flash_attn auto-install picking the oldest wheel, leaving newer GPUs without a kernel (#289)**
  - When auto-selecting a wheel, score ties fell back to alphabetical order and preferred the oldest version (e.g. 2.6.3), which has no matching kernel on newer GPUs (e.g. 5090 / Blackwell sm_120) — causing flash_attn to fall back entirely and spam the misleading "only supports Ampere GPUs or newer" message.
  - After the fix, score ties favor the newer version, and both the auto-install and the candidate list put the newest first.

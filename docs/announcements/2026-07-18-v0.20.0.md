Krea 2 模型族全流程支持 · fp8 底模训练与推理 · 显存策略与编排 · 模型来源统一管理 · 预设自动保存

### 新增

- **Krea 2 模型族：训练到出图全流程（#408–#425）**
  训练配置新增「模型族」选择：除 Anima 外现在可以训 Krea 2（12.9B 单流 MMDiT，Qwen3-VL 文本编码）——Raw 底模 LoRA 训练、训练中采样、正则集 AI 生成、训练后评估、测试出图（含 Turbo 8 步蒸馏出图）全链路可用；输出 LoRA 与 ComfyUI 直接互通（采样组合 Euler + Simple 逐字对应）。入门与训练 tips 见公告「Krea 2 支持上线」。

  - 下载中心按模型族分区，Krea 2 的 Raw / Turbo 各有 bf16 与官方 fp8 版，文本编码器 bf16 / fp8 单选；VAE 与 Anima 共享不重复下载
  - 切换模型族是确认式动作：弹窗逐项列出将重算的权重路径与族默认值，确认才生效；参数下拉按族过滤选项，Anima 专属功能（NaViT、tag 打乱等）对 Krea 2 自动隐藏
  - 训练开始默认把全部 caption 预编码进文本缓存、随即释放文本编码器（省约 9 GB 显存），大显存云端可关闭改为常驻在线编码
- **fp8 底模：训练与推理全线（#426, #434, #435, #436, #437, #444）**
  官方 fp8 权重现在既能当训练底模也能当推理底模：选中 fp8 文件即生效，无需额外开关。训练侧（fp8_base）权重显存 25.6 → 13.1 GB，Krea 2 训练从 32 GB 下探到 24 GB 级显卡；推理侧 fp8 常驻显存约减半，LoRA 以 merge 回写方式挂载、与 ComfyUI 出图逐位一致。

  - XY 矩阵在 fp8 底模上支持 LoRA 强度轴（逐格重 merge）；LoHa 与 rs-LoRA 均可挂 fp8 底模，DoRA 暂不支持
  - 文本编码器官方 fp8 版（5.2 GB）训练与出图全线可用，只下这一份即可；切换精度自动重建文本缓存
- **显存与内存编排（#427, #437, #438, #439）**
  测试出图新增「显存策略」三档（默认·用后释放 / 省显存 / 性能优先）：任务开始把全部 prompt 预编码进缓存后释放文本编码器，采样期文本编码器零显存占用——16 GB 卡在「省显存」档可跑 Krea 2 fp8 出图；评估出图与正则 AI 生成同样接入。训练与出图加载大权重前有内存护栏：内存不够时给出可操作报错，而不是整机换页卡死。「清理显存」现在能真正清干净（补清 cuBLAS 工作区残留）。
- **civitai 社区 LoRA 直接加载（#426, #428）**
  PEFT / comfy 键格式的 Krea 2 社区 LoRA（civitai 生态常见形态）现在可以直接在测试页挂载，bf16 与 fp8 底模都支持，无需转换。
- **模型来源统一管理（#446）**
  WD14 / CLTagger / 评估指标 / 放大器 / 训练主模型五处的「换模型」统一为同一种候选列表：内置候选 + 你添加的候选，底部「添加下载」「添加本地文件」双入口；本地文件候选只会从列表移除、永不删你的文件。旧配置自动迁移，选中项不变。
- **LLM 打标预设统一编辑器（#445）**
  LLM 打标器的预设编辑收口到一个全局编辑器（设置页与打标页打开同一个），字段不再两处不一致；设置页 LLM 区改为预设管理列表（单选默认 + 新建 / 编辑），打标页只留本次运行的预设选择；在设置里改完参数，打标页立即生效。
- **生成页：底模 / 文本编码器选择持久化 + token 计数（#441）**
  测试页选过的底模与文本编码器现在跨页面、跨会话记住（不选则跟随全局设置）；prompt 框右下角显示 token 计数；在线出图不再把超过 512 token 的 prompt 静默截断。
- **生成任务超时兜底（#443）**
  设置里可配置生成任务超时（默认不开启）：整机卡死等极端情况下到时强制终止推理进程并标记失败，下个任务自动重新加载，不再需要重启电脑。

### 变更

- **新配置的默认 LoRA 类型改为经典 LoRA（#442）**
  新项目与新预设的 `lora_type` 默认值从 `lokr` 改为 `lora`；已有配置显式落盘的值不受影响。
- **下载中心已下载资产的操作改为「删除」（#437, #442）**
  「重新下载」的隐式覆盖退役：先删除、再下载；打标 / 评估 / 放大器模型区同步接入删除。
- **配置互斥规则双端强制 + 联动确认（#431, #432）**
  界面上声明过的参数互斥现在后端同样强制：手写 yaml / 命令行的非法组合从静默放行改为启动即报错并给出原因；在界面里触发联动改值时（如开 NaViT 联动 attention backend）先弹确认框列出将改动的字段，确认才应用。配置 YAML 预览与实际落盘现在物理同源，所见即所存。

### 改进

- **预设页自动保存 + 三页统一保存指示（#433）**
  全局预设页与训练 / 设置页一致改为自动保存（600ms 防抖），编辑不再需要点保存；训练、预设、设置三页统一「✓ 已保存」指示；禁用状态的输入框现在有明确的灰显。

### 修复

- **全局换选模型后，旧版本的训练配置不跟随（#439）**
  设置页换选主模型 / 文本编码器后，已有版本开启「自动同步路径」时训练配置此前仍显示并使用创建时的旧路径；现在读取与训练入队都跟随全局选择。
- **训练任务日志看不到 step 进度（#439）**
  Studio 里跑的训练任务日志此前可能整个 epoch 一行 step 进度都不出（输出缓冲 + 进度条在非终端环境刷屏）；现在按行实时输出。
- **Windows 上取消任务可能残留训练进程（#410）**
  进程终止命令因权限等原因失败时，此前任务显示已取消但进程仍在占用 GPU；现在自动回退到逐进程递归终止。

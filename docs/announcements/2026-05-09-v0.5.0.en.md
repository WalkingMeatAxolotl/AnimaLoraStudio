Test image generation + prior generation + Setup rewrite + Settings split + CLTagger (49 commits / 132 files)

### Added

- **Generate page for test images + XY matrix evaluation (standalone tool page, persistent inference daemon) (#19, #22)**
  - "Generate" entry in the sidebar; `/api/generate` + `runtime/anima_generate.py`
  - Inference daemon (stays resident on the GPU so it doesn't reload every time)
  - XY matrix evaluation (parameter sweep)
  - `inference_core` extracted; fixed a P0 bug in multi-LoRA loading
  - SSE now shares a single EventSource, fixing the hang on viewing outputs / reloading the page
  - Random favicon rotation (noal_*.png)
- **AI-generated priors (use the base model with no LoRA to generate images as a regularization set) — added a prior tab to Step 4**
  - Added a "prior generation" tab + explainer to Step 4
  - `/api/projects/.../reg/generate-prior` + `runtime/anima_reg_ai.py`
  - `RegMeta.generation_method` distinguishes manual vs. AI-generated
- **Resume training: pick an existing training state / LoRA checkpoint directly in the UI instead of typing a path**
  - A "📁 Browse this project" button next to the `resume_state` / `resume_lora` fields: pops a dropdown anchored to the field, listing every available file in the project grouped by version. You see semantic labels like "baseline / step 2476" instead of deep `studio_data/projects/.../output/...` paths
  - Selecting one writes the absolute path back to the field (the field value is still the real path, so the backend protocol is unchanged)
  - For external files or a checkpoint from another project, just type the path into the field directly (no picker pops up, leaving a blank escape hatch)
  - Backend: `versions.list_project_state_ckpts()` / `list_project_lora_ckpts()` + `/api/projects/{pid}/state_ckpts` / `/lora_ckpts` endpoints
  - Fixes the root UX problem: previously you had to dig five levels deep from REPO_ROOT to `output/training_state_step*.pt` just to resume
- **Setup rewrite: GPU-aware torch on first install + venv stale check + --reinstall as a lifeline**
  - `studio.bat` is pure ASCII now (no more crashes under cp936 cmd.exe) + a test as backstop
  - bootstrap: prefers `py -3` on Windows, iterates version checks on Linux
  - venv stale check + `--reinstall` flag (environment lifeline)
  - GPU-aware torch on first install; big warning when a CPU-only build gets installed by mistake
  - Defers the torch reinstall to the launcher process, fixing Windows file locks + self-healing zombie directories
  - Added a PyTorch section to Settings for one-click reinstall to the CUDA build
  - `studio.sh --mirror` flag + configurable HF mirror endpoint (Settings UI toggle)
  - ONNX CUDA errors auto-fall back to CPU during inference; skips the torch wheel preload when CUDA is system-provided
- **Attention backend as a single field (xformers / flash_attn / none), pick one of three (#21)**
  - The single `attention_backend` field replaces the two `xformers` / `flash_attn` bools
  - `/api/xformers/{status,install}` + an xformers card in Settings
  - One-click flash_attn wheel install + a model-layer fast path + a CLI entry point
  - `detect_env` now reads the cuda_tag from the torch ABI instead of relying on nvidia-smi
- **Added the CLTagger tagger (external contribution) (#14)**
  - New CLTagger (external contribution)
  - Extracted `OnnxTaggerBase`, so CLTagger gets the PP10 thread pool for free
  - Tagger registry + a unified `<name>_overrides` persistence key

### Changed

- **Settings split into 4 tabs (Dataset / Tagging / Training / Interface) + ONNX as its own section**
  - Split into 4 tabs: Dataset / Tagging / Training / Interface
  - ONNX Runtime broken out into its own section
  - WD14 / CLTagger switched to the Anima base model styling (radio + inline download)
  - Field alignment + 2K-screen whitespace fix
- **Training page progress bar hidden by default (everything goes through the monitor view)**
  - Training scripts moved to `scripts/` + `tools/`, retiring `monitor_smooth.html`
  - `LoraEntry` extracted into `schema.py` (wrapping up PR-9)
  - Hid the "Monitoring & progress" group; `no_progress` now defaults to True

### Fixed

- **Removed the standalone "Export dataset" button in the collapsed state to avoid accidental clicks**
- **A round of CLTagger UX fixes (#14 follow-up)**

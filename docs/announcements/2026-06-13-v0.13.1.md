数据集大文件上传性能修复 + 测试出图临时缓存加密

### 安全

- 测试出图临时缓存改加密落盘，避免云训练时被扫盘识别（#273）
  - 部分云厂商会扫盘按文件名 / PNG header / 内容分类抓敏感图。本版起测试出图临时缓存（不开自动落盘的那一支）改为加密落盘，文件无后缀、首字节为高熵随机字节，扫盘工具识别不出是图。
  - 加密密钥只在服务器进程内存里生成，进程退出后即丢失；下次启动自动清理上一次的残留文件，即使断电 / 强杀也不会留可读副本。
  - Settings 里开启「测试图片自动落盘」的明文 PNG 路径不变（用户主动选择保存的图本就应该可见 / 可拖去 ComfyUI）。

### 改进

- 数据集上传进度在底部日志抽屉实时显示（#273）
  - 之前上传只能看到进度条停在 100% 然后无反馈等待；现在 zip 内每 25 张 / 5 秒触发一行进度，跟下载任务并列在底部抽屉里。
  - 单张图处理超过 1 秒会额外 emit 一行 `[slow X.Xs: 文件名]`，方便定位拖后腿的素材。

### 修复

- 数据集上传大文件由 40 分钟降到 1-2 分钟 + 上传期间不再卡死页面（#273）
  - 上传 1GB 含数百张图的 zip，原来需要 40+ 分钟，现在 1-2 分钟。booru 下载下来已经是 PNG 的图不再被无谓地解码再重新压缩；JPG 等真需要转 PNG 的也省 5-10× 时间。
  - 上传处理期间页面 F5 刷新会"完全卡死"的问题修了：服务器在处理上传时仍能正常响应其它请求，刷新 / 切别的项目都立即响应。
  - 上传期间在服务器终端按 Ctrl-C 没反应、只能关窗口的问题修了，现在 Ctrl-C 立刻生效。
  - 大 zip 不再一次性整个读进内存，1GB zip 内存占用从 1GB 降到 ~1MB，内存紧的云训练机不再触发系统换页拖慢。
- 测试出图历史栏切路由 / 刷新页面后不再丢失（#273）
  - 之前测试页历史栏（不开自动落盘时）的图，只要切到别的页面或 F5 刷新就消失；现在切走再回来 / 刷新页面都还在，直到 server 重启。
  - 历史索引改成 server 端维护，浏览器零持久化层，不再出现"前端显示但实际拉不到图"的脏数据。

NaViT packed training · Global WandB presets · Configs persist active fields only · Simplified dedup review · CAME optimizer

### Added

- **NaViT / Patch-n-Pack packed training (off by default) (#371, #372)**
  Training gains a `navit_packing` mode: images of different sizes are packed into a single sequence, with each image attending only to its own content. VRAM utilization is governed by a token budget instead of the "one bucket size per batch" constraint.

  - Optional `navit_native_resolution`: images train at their native size (16px-aligned) instead of being resized / cropped to the bucket grid; images over the token budget are scaled down proportionally by default (or can be made to fail instead).
  - Both are off by default; with them off, training behaves exactly as before.
- **Resolution-aware timestep shift (off by default) (#383)**
  New `timestep_shift_resolution_aware` switch in the training config: when enabled, the timestep shift scales automatically with each image's token count relative to the base resolution (higher resolution → stronger shift), so multi-resolution and NaViT native-resolution runs place every image in an appropriate noise range. Images at the base resolution are unaffected.
- **CAME optimizer (#358)**
  New CAME optimizer option: factorized row / column second moments reduce optimizer-state VRAM, with confidence-guided weighting to suppress the approximation noise the factorization introduces. Six advanced parameters (betas / eps / clip), defaulting to the paper's values.
- **LLM tagging can inject local tagger output (#230)**
  LLM presets gain an "assist tagger" option (off / WD14 / CLTagger): each image is pre-tagged by the local tagger first, and the result replaces the `{{tags}}` placeholder in your prompt as a prior to guide the LLM's caption.

  - Two built-in assist prompt presets ship with `{{tags}}` included (structured caption / single-line natural language); you can also add `{{tags}}` to any preset's prompt yourself. It only takes effect when the prompt contains `{{tags}}`.
- **YAML preview on the training config and presets pages (#366)**
  Both pages can now show the effective config exactly as it will be written to disk and used in training. The presets page's old "TOML preview" was misleading (the fields were never sd-scripts format) and has been replaced by the YAML preview.

### Changed

- **WandB settings moved to global Settings; WandB fields removed from training configs (#367)**
  WandB moves from a per-config override block in the training config to preset-based management in global Settings (Settings → WandB): your API key is no longer written in plain text into config.yaml, task snapshots, exported presets or bundles. Existing global settings migrate automatically; if you ever filled in WandB inside a training config, see the pinned migration post (including the API-key rotation advice).

  - Presets support switching between multiple setups, save-as, and import / export (exports contain the real key — handle with care).
  - The artifact upload settings moved out of the sample-image upload subsection (they were unrelated) and merged into a three-way choice: off / keep latest only / keep all.
  - The `--wandb_*` CLI flags are gone along with the fields; use `WANDB_*` environment variables instead.
- **Training configs persist active fields only (#366, #369)**
  config.yaml, presets and task snapshots no longer include sub-parameters of disabled features or internal defaults — files are much shorter and easier to read. Existing files load unchanged; missing keys fall back to their defaults.

  - This also fixes a real issue: leftover non-default values in hidden fields (e.g. sub-parameters of a previously selected optimizer) could silently affect training. They are now pruned at write time — what you see is what trains.
- **Dedup review simplified to two scan parameters; crops merged into groups (#381)**
  The dedup scan goes from 18 parameters down to two: match scope and sensitivity (loose / standard / strict), with the remaining thresholds fixed as built-in constants. Crop / resize relations are no longer a separate "quality candidates" tab — they join the regular duplicate groups as stricter duplicates, and are only detected under the "duplicates + near-duplicates" scope.
- **Manage tagger models right on the Tagging page (#378, #380)**
  The Tagging page's right panel becomes a "current tagging status" card (tagged count / method / model / trigger word / images this run will process); model download, selection and download-source switching move into each tagger's "advanced parameters" section — no more round-trips to global Settings.

### Improved

- **Sidebar version picker rebuilt as a version row (#384)**
  The sidebar "training version" block becomes a compact version row: switching opens a popover (no longer stretches the sidebar), and hovering reveals create / export / delete. The Overview page's version rail gains the missing delete action, so all four version actions match in both places.
- **Regularization page layout rework (#379)**
  The "Generate" tab is now split into two columns: generation parameters on the left, regrouped into "reg-set settings / image settings", and a new status panel on the right whose "images this run will generate" figure updates live from source / mode / target count, matching the backend's math.
- **Unified actions across preprocessing pages (#381)**
  Tool switching becomes an underline tab row attached to the header, and each page's primary actions move to the top-right of the header. The crop page drops its parameter sidebar to give the workspace more room; "smart clustering" is renamed "auto clustering" and moves into a header popover.
- **Model root migration can merge into existing data (#368)**
  Migrating the model root in Settings no longer refuses when the target already has data: choose "skip existing files" (copies only what's missing — ideal for pointing at a directory shared with other tools) or "overwrite existing files". Copies are atomic per file, so a failure mid-way never corrupts what's already in the target.
- **Queue row actions in their proper place (#370)**
  "Resume training" and the paused-row "resume / cancel" buttons move from the time column into the trailing action column as icons; resume now asks for confirmation, so a stray click can't kick off training.
- **Dataset preview in the prompt picker zooms on click (#375)**
  The dataset image preview at the bottom of the Generate page's "pick prompt from training set" panel now opens full-screen on click, for side-by-side comparison with generated results.

### Fixed

- **Mid-step generation previews no longer show inverted colors (#377)**
  In-progress previews decoded latents with the wrong decoder (Flux instead of the Wan2.1 latent space), producing inverted / garbled colors at a tiny size. They now use the correct projection, with normal colors, filling the result area.
- **Clicking "Generate" while viewing history returns to the live view (#376)**
  Clicking "Generate" while looking at an older image (especially XY grids) used to leave the result area stuck on that old image; it now returns to the live view so you see the new task's progress immediately.
- **HTTPS traffic no longer bypasses the proxy when only one field is set (#374)**
  If either the HTTP or HTTPS proxy field is empty, the other is now used for both (as the settings page always claimed but never implemented): with only an HTTP proxy configured, booru queries and downloads no longer bypass it and time out. With both fields set, each is respected independently.
- **Square images no longer land in an undersized bucket (#313)**
  When several candidate buckets tie on aspect-ratio distance (e.g. multiple 1:1 buckets at base 1536), the bucket closest in area to the base resolution now wins; previously the first generated smaller bucket won (1472×1472 instead of 1536×1536). The frontend bucket preview mirrors the same rule. Affected datasets will bucket slightly differently after upgrading.

### Removed

- **"Blur candidates" removed from dedup review (#381)**
  Blur detection is quality filtering, not deduplication; the tab and its scan logic are gone.

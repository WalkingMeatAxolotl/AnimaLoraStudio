One-click webui self-update + training-stack decomposition (ADR 0002 / 0003)

### Added

- **Support for the Anima 1.0 base model (latest now points to 1.0 by default) (#61)**
  Upstream `circlestone-labs/Anima` released the official 1.0 version (`split_files/diffusion_models/anima-base-v1.0.safetensors`).

  - `ANIMA_VARIANTS` gains a `"1.0"` entry; `LATEST_ANIMA` switches to `"1.0"`
  - the dict order is changed to "latest first" — so the `find_anima_main` fallback and the variants list `build_catalog` hands the UI follow the "newest first" intuition (previously [LATEST] + [dict insertion order] would hit the oldest variant first when LATEST wasn't on disk)
  - the defaults in `schema` / `secrets` / `server` / `runtime/training/cli` / the front-end Settings all switch to 1.0 — **this affects only fresh installs and users who've never written secrets.json**; in existing versions' yaml, `transformer_path` is an absolute path that stays put, preserving training reproducibility
- **One-click webui self-update: a dual-channel upgrade panel + rollback + a dev channel (ADR 0002) (#51, #52, #53)**
  Implements ADR 0002: do the whole git pull / restart / rollback flow from Settings → System → the version card, without going back to the command line.

  - **Approach A** (flag + shell wrapper loop, modeled on A1111 / SwarmUI): a `tmp/restart` flag + a `cli.py` inner loop + a `studio.sh / studio.bat` wrapper loop + `POST /api/system/restart`. Hard constraint: update / restart / rollback are refused while a task is running
  - **Main path**: `studio/services/updater.py` + four endpoints (`/version` `/update_check` `/update` `/preflight`) + a startup `apply_pending()` git reset + incremental pip / npm install + a Topbar update badge
  - **Dual-channel upgrade panel** (#52 redesign): side-by-side master / dev cards + container-query responsiveness + channel badges (stable/green · dev/orange) + an inline preview panel replacing every dialog modal; CHANGELOG.md is parsed and embedded in the card; the dev card's commit timeline lets you click any commit to switch to it
  - **Four pre-flight checks**: dirty working tree / running tasks / requirements.txt diff / `.last_version` preview; any error disables the confirm button
  - **Rollback + failure banner**: `.update_status` / `.last_version` / `.update_log` + `/api/system/rollback` + a red failure banner inside the master card + a "view full log" modal
  - **Dev channel toggle**: `SystemConfig.show_dev_channel` is persisted; the automatic check only looks at master (so developers aren't constantly nagged by a badge from frequent dev commits), and dev must be triggered manually
  - **Installer self-check (exit 42 protocol)**: `cli.py` / `studio.sh` / `studio.bat` are compared by sha256; if any changes → cli.py keeps the flag and exits 42, and the wrapper, seeing exit 42 + tmp/restart, re-execs itself
  - **Historical versions shown as tags** (within #54): `updater.exact_tag_for(sha)` uses `git describe --tags --exact-match`; on a hit the rollback button shows `v0.6.0`, otherwise it falls back to sha[:8]
- **Internal refactor of the training stack (no change to user-facing behavior; makes it easier to add custom adapters / schedulers later) (ADR 0003) (#56, #57, #58)**
  Implements all of ADR 0003: `runtime/anima_train.py` is split from a 2901-line mega-script into the `runtime/training/` sub-package (25 files, a 128-line thin entry) + four plugin registries + an `AdapterProtocol` hook. **Training behavior is byte-for-byte equivalent** — every hook on the LyCORIS path is a no-op, and the optimizer / scheduler go through the same kwargs via a build wrapper.

  - **PR-A (#56) module split**: 12 modules — bootstrap / observability / model_loading / models / text_encoding / state / dataset / sampling / cli / timestep_sampling / noise / loss_weighting. The sister scripts (anima_daemon / anima_generate / anima_reg_ai) keep working with zero changes thanks to a re-export contract
  - **PR-B (#57) main() split into phases**: the 793-line main() → six phase functions (bootstrap / models / dataset / optimizer / resume / finalize) + one train_loop, wired together by a `TrainingContext` dataclass (43 fields + 3 methods). The three near-75-line duplicate sample blocks in main() are folded into a `run_sample` helper
  - **PR-C (#58) plugin registry + AdapterProtocol**: four plugin sub-packages (adapters / optimizers / schedulers / inference_samplers) + explicit BUILDERS dicts; three if-elif dispatches are replaced with registry calls. AdapterProtocol carries three optional hooks (on_step_begin / regularization_loss / excludes_weight_decay) to reserve room for future T-LoRA / OFT / AdaLoRA
  - **Automatic schema↔registry consistency check**: three plugin sub-packages expose `validate_schema_consistency()`, which bootstrap_phase runs once at startup; a missing registration or missing schema fails early
  - **Steps to add a new variant** (see [`runtime/training/README.md`](runtime/training/README.md) + ADR 0003 Cases 3-6): write `training/{plugin}/{variant}.py` with a build function + add one line to the BUILDERS dict + add a value to the schema Literal, with zero changes to phases / loop / main
- **Training-stability extensions: NaN skip + new noise / loss / timestep sampling options + cross-attn KV trim (#55)**
  Cherry-picked from PR #49 (saltysalrua); after review we kept 5 low-risk, high-value commits plus 4 hardening changes of our own. **T-LoRA / the Ortho-Hydra adapter / manual OrthoGrad don't go into the main repo** — they live in the long-term `experimental/pr49-adapters` parking lot.

  - **ProdigyPlus upstream version-compat filter** + `eps=None` support + StableAdamW (fixing a TypeError from upstream API drift)
  - **NaN detection**: skips a step when loss / grad is non-finite, so an occasional bf16 + Prodigy spike no longer blows up the whole run
  - **Timestep-dependent loss weighting**: `min_snr` / `detail_inv_t` / `cosmap` + `weight_cap_ratio`
  - **Configurable timestep sampling**: `logit_normal` / `uniform` / `logit_normal_low` / `mode` + shift
  - **noise_offset + pyramid_noise** noise augmentation
  - **cross-attn KV trim**: surgically split out of `c5e81c2`, taking only the kv_trim part (dropping the T-LoRA changes); also fixes a `_bucket = 512` fallback (the original code raised a NameError when `_actual > 512`)
  - **Dead T-LoRA dispatch cleanup**: removed a hidden ImportError landmine leaked in by the original commit
  - **Cluster prefixes added to 9 new fields' descriptions**: [Noise augmentation] / [Timestep sampling] / [Loss weighting] / [Performance], so the tooltips show which group each belongs to
  - **Allowlist added to `_filter_kwargs_by_signature`**: when upstream drops a schema-exposed ppsf_* field, it now raises explicitly instead of silently logging (so a setting you checked doesn't quietly become a no-op only discovered after an 8-hour run)
- **New Prodigy+ScheduleFree optimizer (solves the occasional "style-jump epoch" problem during Prodigy training) (#46)**
  Prodigy's internal `d` estimate can "jump a gear" under the triple noise of Flow Matching timestep randomness + a small dataset + LoRA's low parameter count — `d` is a non-decreasing accumulator, so once an anomalous batch pushes it up a gear, the rest of the run uses a larger effective step size. Community research found that Flux / Qwen-Image / HiDream / video DiT LoRAs have already made PPSF a de facto default (ai-toolkit / SimpleTuner / sd-scripts all integrate it).

  - **Name**: `prodigy_plus_schedulefree` (the full snake_case name, to avoid clashing with a future vanilla Prodigy)
  - **eval/train switching**: an `optimizer_eval_mode()` context manager, which is harder to half-forget than a helper pair; every `injector.save` is inside the ctx → the saved .safetensors holds the averaged weights x, ready to use
  - **Three layers of scheduler mutual-exclusion defense**: (a) front-end disable + auto-reset → (b) a pydantic model_validator → (c) a SystemExit at anima_train CLI startup
  - **Dependency pinned to >=2.0.0**: PPSF v1.9.2 ↔ v2.0.0 state_dicts are incompatible
  - **New schema metadata `disable_when`**: reuses the `show_when` expression syntax; SchemaForm implements disabled + hint + forcing the value to its default

  - **Applied the tooltip treatment**: the ServiceSection restart note / WD14 / CLTagger / anima_main model-card descriptions / the xformers mutual-exclusion note / 7 long Layer 1 descriptions (hf endpoint / wandb throttling, etc.) all move from inline `<p>` / `desc=` to a ⓘ next to the label / title

### Improved

- **Training page: inline preset creation + drag-to-reorder tag chips + a renamed "export training set" button (#47)**
  Four independent small polish items in one PR:

  - **Inline preset creation**: the training page's picker grid gains a dashed "+ New preset" card that switches to an inline SchemaForm (the name defaults to `<slug>_<label>`, and the description is stored in localStorage). Previously you had to leave for `/tools/presets` to create one and then come back
  - **Drag-to-reorder tag chips**: dnd-kit PointerSensor + a 6px activation distance → dragging doesn't conflict with clicking × to delete; `addTag` now appends to the end (consistent with the drag mental model)
  - **"Export to CNB" → "Download training set"**: the button text / toast / title / function name (`exportForCnb` → `downloadTrainZip`) all drop the CNB binding
  - **Optimizer description cleanup**: removed the "needs pip install prodigyopt" wording (both packages are already in requirements.txt)
- **Dialogs unified to use the app's own styling (no more native browser confirm/prompt) + topbar/sidebar polish (#48)**
  - **`useDialog()` hook**: imperative, promise-based confirm / prompt / alert in `src/components/Dialog.tsx`; tone (default / danger / warn) controls the confirm button's color; ESC + clicking the overlay = cancel; prompt includes synchronous validation
  - **22 replacements**: Train / Settings / Queue / Projects / Layout / Presets / SaveBar / Regularization / Curation / Download; dangerous actions (irreversible deletes) use danger, and big actions (system-level things like installing packages) use warn
  - **topbar search icon moved to the far right**: it no longer jumps around when a training task pushes the pill
  - **the sidebar no longer auto-collapses when entering a project**: manual collapsing is persisted via sessionStorage
  - **selecting a version on overview jumps to download + reuses NewVersionDialog**: removes `window.prompt`, supports fork-from + auto-activate

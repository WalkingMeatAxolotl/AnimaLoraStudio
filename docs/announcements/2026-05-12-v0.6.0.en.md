LLM tagger + training-monitor observability + reorganized Settings layout

### Added

- **New LLM tagger (a second tagger that produces long captions via an OpenAI-compatible API) (#18, #34, #35)**
  - Works with any OpenAI Chat Completions-compatible endpoint — OpenRouter, vLLM, Ollama, and others
  - Training WandB integration: `tracker_project` / `tracker_run_name` / `wandb_api_key` are wired through to sd-scripts, and the run URL plus key metrics are posted back to the project page
  - Opt-in by default: the LLM isn't called unless you configure an endpoint, and WandB is off until you explicitly enable it
  - JoyCaption becomes a builtin preset; `JoyCaptionConfig` is removed
  - Prompt → message sequence: an `LLMMessage` type with dnd-kit drag-and-drop editing, supporting multi-turn / few-shot conversation formats
  - Settings UI: a two-column grid, four sections merged into one large card, and a full-height composer
- **Topbar now shows live CPU / GPU / memory / VRAM usage (#37, #42)**
  - Four equal-width pills in the topbar (CPU / GPU / MEM / VRAM, min-width 96px), justified across
  - Pulled from `nvidia-ml-py` (pynvml is unmaintained); the backend `_StatsThread` pushes to the frontend every 2.5s over SSE `system_stats_updated`
  - Monitor switched to an incremental protocol: step-wise deltas replace a full snapshot every second, cutting a 10k-step training payload from O(N) to O(1)
  - Cold start loads the full history: `/api/state` defaults to `max_points=0` (no downsampling), and the frontend `MAX_LOSSES / MAX_LR` went from 5000 → 50000 to match the backend `train_monitor`
  - `_SelectiveGZipMiddleware`: a 10k-step `/api/state` drops from ~500KB to ~100KB (5x), while `/api/events` SSE and `/samples/*` images are whitelisted and skipped
- **ModelScope added as a model download source (#25)**
  - Switch to ModelScope when a download won't come through from HF; Settings gains a `download_source` option, still defaulting to `huggingface`
  - `_get_download_source()` priority: `MODELSCOPE_SOURCE` env > secrets > `'huggingface'`
  - Models with a mapping download via the ModelScope CLI; those without one fall back to HF automatically
  - If onnxruntime fails to install, it falls back to the Tencent PyPI mirror (fallback only — the default source is unchanged)

### Changed

- **Settings reorganized: new Monitoring tab + breadcrumb navigation + sticky anchors (#36)**
  - New "Monitoring" tab, with WandB moved over from "Training"
  - HF / ModelScope are merged into a "Model download source" section under "Training", rendered conditionally on `download_source`
  - PageHeader gains a `tabs` slot; the eyebrow is removed everywhere (it duplicated the topbar breadcrumb)
  - The topbar breadcrumb is now a React-Router `Link` you can click to navigate
  - A sticky section index on the right: `IntersectionObserver(rootMargin: -20% 0px -70% 0px)` highlights the active section with smooth scrolling
  - The LLM tagger's sampling parameters and image preprocessing are merged into an "Advanced parameters" panel that's collapsed by default
  - URL routing is unchanged (`/tools/settings`); tabs are React state, so old browser bookmarks keep working

### Improved

- **Queue output page now has direct-link downloads + batch zip + sorting by step/seed (#33)**
  - Queue detail page: direct-link downloads, batch zip, sorting by step / seed, and consistent file naming
  - Previously you had to dig down into `studio_data/projects/.../output/` to get your training artifacts

### Fixed

- **LLM tagger follow-ups: caption dedup + WandB off by default + sample thumbnails (#34)**
  Four P0 follow-ups from PR #18:

  - **Duplicate captions**: `utils/caption_utils.py` and `studio/services/caption_format.py` each had their own `normalize_caption_json`, with subtly different merge logic (lowercase dedup vs. a plain extend). Now a single source is re-exported, so the two can't drift apart when the schema changes
  - **WandB off by default**: PR #18 defaulted `log_with_wandb=True`, forcing everyone to install wandb just to train; it's now opt-in
  - **Training sample thumbnails**: the original 1024×1024 PNGs were sent straight to the frontend; the backend now scales them down to 256px thumbnails
  - **Custom `output_format`**: previously hardcoded to PNG, now a field you can set
- **Danbooru estimate API 403 (missed in the 0.5.2 hotfix, which only fixed search) (#41)**
  - The 0.5.2 hotfix only fixed search; the estimate call goes through a separate path that was still sending a bare UA. This adds the `AnimaLoraStudio/<version>` UA + `Accept: application/json` there too
  - A matching estimate regression test was added in `tests/test_downloader.py`
- **Fixed a 500 error when generating AI priors, caused by a wrong variable name (in #42)**
  `reg_generate_prior` writes its config using `STUDIO_DATA / "reg_ai_configs"`, but the `from .paths import (...)` at the top of `server.py` was missing `STUDIO_DATA`, so the route crashed the moment it was called. Fixed with a one-line import.

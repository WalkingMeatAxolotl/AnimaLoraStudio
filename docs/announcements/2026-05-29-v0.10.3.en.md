Added a global HTTP/HTTPS proxy setting (covering the booru download path)

### Added

- **Settings supports a global HTTP/HTTPS/SOCKS5 proxy, covering booru download requests (#162)**
  Settings → Tools adds a "Global proxy" section. Once enabled, the HTTP/HTTPS/SOCKS5 address you enter is injected into download requests for the booru image sources (Gelbooru / Danbooru), so networks that can't reach these two sites directly can still scrape images. An extra `no_proxy` field takes comma-separated exception addresses (e.g. `localhost,127.0.0.1`), and matching requests bypass the proxy. Coverage is currently limited to the booru download path; routing model repositories (HuggingFace / ModelScope) through a proxy still requires the `HF_ENDPOINT` mirror or OS-level `HTTP_PROXY` / `HTTPS_PROXY` environment variables.

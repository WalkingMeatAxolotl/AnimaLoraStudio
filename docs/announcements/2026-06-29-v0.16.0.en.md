In-app announcement center · app moved back to the root URL · instant-apply settings · faster Generate page · LoRA evaluation (experimental) · several training and UI fixes

### Added

- **In-app announcement center — a bell in the top-right for updates and migration notes (#336, #337, #339).**
  Until now nothing told you what changed in a release or warned you about behavior changes. An announcement now pops up once after each update, and the bell is always there to open.

  - Version list on the left, post body on the right, with full markdown (headings / lists / links / code).
  - The changelog moved from "dig through Settings" into the announcement center, and every past version now has an English version (English users no longer fall back to Chinese on older releases).
  - Posts are grouped into changelog / notice / migration; behavior changes that need your attention (like this release's URL move) are pinned separately.
- **LoRA training evaluation (experimental) — automatic quality metrics on checkpoints after training (#138, #319, #322, #323, #324).**
  Landed as the foundation for upcoming evaluation work. The current metrics have limited discriminative power on anime-domain outputs and are for reference only; they are opt-in and not the focus of this release.

  - After training, eval samples are generated and scored with CLIP-T / CLIP-I / DINO-I, plus anime-domain tag recall and character consistency (CCIP), compared against a base-model baseline to isolate the LoRA's net gain.
  - You can hand-pick a validation set per project; eval progress and logs feed the shared task log drawer.
- **Editable project slug on creation, and selected project kept across pages (#333).**
  All-Chinese project names used to collapse to `project` on disk and got hard to tell apart; picking a project and switching to Generate / Queue also dropped the selection.

  - The new-project dialog adds a "project slug" field that validates live and previews the final on-disk name (leave it blank to derive from the title as before).
  - The selected project stays in the sidebar when you move to global pages like Queue or Generate — no need to reselect from the list.
- **Image counter ("N / M") at the bottom of the large-image viewer (#326).**
  When browsing large images with ←/→ across Download / Preprocess / curation / regularization / training samples / XY grid, the viewer now shows your position in the sequence.

### Changed

- **App moved back to the root URL; the `/studio/` prefix is gone (#334).**
  The address changes from `localhost:8765/studio/` to `localhost:8765/`, which also fixes deployments that reverse-proxy the container port to root (e.g. ModelScope / HuggingFace Spaces) failing to load.

  - Old `/studio/...` links still redirect automatically this release and will be removed in the next one — update your bookmarks.
  - See the pinned "App moved to the root URL" migration post for details.

### Improved

- **Global Settings is now instant-apply; the bottom save bar is gone (#340).**
  Toggles and dropdowns save on change; text and number fields save on blur or Enter. A save indicator sits in the bottom corner — no more "Save" button.

  - Destructive actions (delete a preset, restore dictionary defaults, remove a custom local model) ask for confirmation first.
- **The Generate page opens noticeably faster (#327, #328).**
  Two causes of slow loads with many history images and many projects / versions are fixed.

  - The history rail is now virtualized and reads PNG metadata straight from the text chunk (local test: history scan 15.4s → 0.12s).
  - The LoRA list loads lazily — opening the page no longer fires that batch of project / version requests unless you open the LoRA picker.
  - XY results freeze once generation starts; changing an axis mid-run only affects the next run instead of corrupting results already produced.

### Fixed

- **Fixed fp16 dtype mapping and GradScaler for fp16-only GPUs like the V100 (#335).**
  Corrects the fp16 dtype mapping and GradScaler persistence so cards that only support fp16 (e.g. the V100) train correctly.
- **VAE tiling logs during the cache stage no longer spam or mislead (#332).**
  During latent caching, every image logged a tiling decision (200 images, 200 lines) reading "used > total VRAM" as if memory were exhausted. The tiling itself is correct (it avoids the VRAM paging cliff); this release de-duplicates and demotes those logs.
- **Added a cancel button for AI-prior generation, made the settings drawer dismiss on sidebar click, and stabilized the version panel on load (#329).**

  - The regularization "AI prior" run now has a cancel button (previously only booru scraping did).
  - The settings drawer now dismisses when you click the sidebar, not just the main area.
  - The version "what's new" panel no longer jumps in height while loading or switching versions.

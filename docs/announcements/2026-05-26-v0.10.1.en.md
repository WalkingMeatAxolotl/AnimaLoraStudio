Hotfix — fixes three silent regressions introduced in v0.10.0

### Fixed

- **JSON caption mode silently fell back to TXT, disabling category shuffle and trigger-word handling (#135)**
  A path miscalculation introduced in v0.10.0 meant that with `prefer_json=True`, training could never find the JSON caption tool files and silently fell back to TXT mode. Affected: training flows that use the tag worker to write `.json` tags for images; under the JSON path, category shuffle (shuffling within character / appearance / action groups), keeping the `meta.trigger` trigger word first and out of the shuffle, and similar features all stopped working. The loss curve kept descending with no errors, but the caption distribution the model actually learned didn't match your settings.
- **The Generate page's generate button wasn't disabled while a GPU task (training, reg-ai, etc.) was running (#135)**
  The Generate page (Studio → Generate) shares the GPU with training, and running both at once causes VRAM contention, stuttering renders, and in bad cases an OOM in the training process. The sidebar previously only carried a "shares the GPU with training" note while the button stayed clickable. Now, when a non-generation task is detected running in the queue, the "Start generating" button is grayed out and hovering shows "queue task #N is using the GPU — wait for it to finish or cancel it"; the button becomes clickable again automatically once the task ends.
- **The task detail "Linked config" tab refetched automatically every 2 seconds during training (#135)**
  On the task detail page, while training was running the "Linked config" tab flickered continuously and re-requested the snapshot config every 2 seconds, stuttering the browser. The linked config is immutable data frozen when the task starts and shouldn't be refetched repeatedly; the fix fetches it just once each time you switch tasks or a task moves from pending to running.

Dataset large-file upload performance fix + encryption for the test-image temporary cache

### Security

- **The test-image temporary cache is now written to disk encrypted, so it can't be identified by disk scans during cloud training (#273)**
  - Some cloud providers scan disks to flag sensitive images by filename, PNG header, or content. As of this release the test-image temporary cache (the branch where automatic saving is off) is written encrypted: the files have no extension and start with high-entropy random bytes, so disk scanners can't tell they're images.
  - The encryption key is generated only in the server process's memory and is lost when the process exits; the next startup automatically cleans up leftover files from the previous run, so even after a power loss or force-kill no readable copy remains.
  - The plaintext PNG path used when you enable "automatically save test images" in Settings is unchanged (images you deliberately choose to save should of course stay visible and draggable into ComfyUI).

### Improved

- **Dataset upload progress now shows live in the bottom log drawer (#273)**
  - Previously the progress bar would sit at 100% and leave you waiting with no feedback; now a progress line is emitted every 25 images or 5 seconds as the zip is processed, shown alongside download tasks in the bottom drawer.
  - Any single image that takes over a second to process emits an extra `[slow X.Xs: filename]` line, making it easy to spot the assets holding things up.

### Fixed

- **Large dataset uploads drop from 40 minutes to 1–2 minutes, and the page no longer freezes during upload (#273)**
  - Uploading a 1GB zip with hundreds of images used to take 40+ minutes and now takes 1–2. Images already in PNG from a booru download are no longer needlessly decoded and re-compressed, and even JPGs that genuinely need converting to PNG are 5–10× faster.
  - Fixed the page freezing completely on an F5 refresh during upload processing: the server now stays responsive to other requests while handling an upload, so refreshing or switching to another project responds immediately.
  - Fixed Ctrl-C doing nothing in the server terminal during an upload (which forced you to close the window); Ctrl-C now takes effect immediately.
  - Large zips are no longer read into memory all at once: a 1GB zip's memory use drops from 1GB to ~1MB, so memory-tight cloud training machines no longer trigger system paging and slow down.
- **The test-image history strip no longer disappears when you change routes or refresh (#273)**
  - Previously the images in the Generate page's history strip (when automatic saving was off) vanished the moment you navigated to another page or hit F5; now they survive navigating away and back or refreshing, until the server restarts.
  - The history index is now maintained server-side with zero browser persistence layer, eliminating the stale state where the frontend showed an image it couldn't actually fetch.

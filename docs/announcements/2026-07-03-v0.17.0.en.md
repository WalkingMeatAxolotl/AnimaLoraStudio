Queue page overhaul · Generation task queueing · Scheduled training · Unified VRAM admission · Tagging output format cleanup

### Added

- **Queue page overhaul: sections, all task types, search / filter / pagination (#343, #348, #359, #361, #362, #363)**
  The queue no longer shows training tasks only: training, regularization-set builds, test generation and evaluation sampling now share one queue, split into "Scheduled / Running / Waiting / History" sections, with paginated history.

  - Search by task, config or project name, and filter by type and status; filters are remembered across visits. Defaults to training tasks only — switch to "All" anytime.
  - Task detail pages now adapt to the task type, with a "View result" shortcut (generation tasks jump to their results on the Generate page, regularization tasks to the reg-set page).
  - New "Data jobs" tab: downloads, preprocessing, tagging and evaluation metric jobs are all visible in one place — expand a row for parameters, errors and live logs, cancel it, or jump back to its original page.
  - Tasks and data jobs now share a single ID space; no more two numbering schemes.
- **Generation task queueing: submit back-to-back, unified timeline, full-phase progress (#355, #348)**
  Clicking "Generate" on the test page no longer interrupts a running job: tasks queue up and run in order, and the right panel becomes a unified timeline (queued / running / done) with per-task cancel and a clear-queue button.

  - New batch count next to "Generate": enqueue N tasks at once, with auto-incrementing seeds.
  - The progress bar now covers every phase — model load, text encoding, sampling and VAE decode — instead of sitting at 0% or 100% for long stretches.
  - Generation task logs stream live in the queue detail page, which also links straight back to the resulting images.
- **Scheduled training (#356)**
  When enqueueing a training run you can now pick an exact start time (e.g. 3:00 AM) or a delay in hours; the task waits and joins the queue automatically. The queue page gets a "Scheduled" section with the planned time, a countdown, and "Start now" / cancel actions.
### Changed

- **VRAM scheduling unified into resource-tier admission (#360, #362, #363)**
  Which tasks may use the GPU at the same time is now enforced by the backend in three tiers: exclusive (training / reg AI prior / test generation / evaluation sampling) runs one at a time system-wide; light (tagging / upscaling / evaluation metrics) may run alongside; downloads are unrestricted. This closes three admission gaps, including one where generation could still run during training.

  - **Tagging, upscaling and evaluation metrics now run alongside training by default** (previously deferred). You can turn this off in Settings; the old `allow_gpu_during_train` switch is deprecated and replaced by `light_tasks_during_train`, without migrating the old value.
  - Exclusive tasks run strictly in enqueue order — no more implicit priority; the queue order you see is the execution order.
  - "Generate" is no longer greyed out while training: submissions simply queue up and start once training finishes.
- **Local tagging output format now follows the output, no manual choice (#353)**
  The output-format selector on the Tagging page is gone: local taggers (wd14 / CLTagger) and LLM text mode always write `.txt` (trigger word first); LLM categorized-JSON mode writes `.json`; images that already have `.json` keep getting `.json` updates.

  - `.txt` captions now honor `tag_dropout` (previously JSON-only); the `keep_tokens` prefix stays exempt from shuffle and dropout, matching kohya semantics.
  - Existing flat-JSON datasets are unaffected — the training side keeps reading them indefinitely.

### Fixed

- **Datasets tagged locally with JSON output no longer train on empty captions (#352)**
  Flat JSON written by the local taggers was misdetected by the training side; the parse failure was silently swallowed and those images trained with empty captions (not even the trigger word). Reading is fixed, and a pre-training check now lists any unparsable JSON captions and refuses to start instead of failing silently.
- **`lora_reg_dims` per-layer ranks were silently ignored for `algo=lora` (#350)**
  Since v0.10.0, training with `algo=lora` + `lora_reg_dims` silently ignored the per-layer ranks (the network trained at the global rank with no warning; LoKr was unaffected). After the fix, retraining with the same config actually applies the per-layer ranks.

  - Note: resuming or continuing an old checkpoint of this kind (uniform rank) on the new version fails with a shape mismatch — drop the `lora_reg_dims` setting or retrain. To check whether a run was affected: its `run.log` shows only `[lora_reg_dims] skipped N modules` and no `applied rank overrides`.
- **"Sample prompts" on the training config page accepts newlines again (#364)**
  The multi-prompt rotation field (one prompt per line) swallowed the Enter key, making multi-line input effectively impossible. Fixed; the three sampling prompt boxes on that page also auto-grow with content now.
- **Announcement center no longer jumps back to the first post (#354)**
  Clicking another post right as the announcement center opened could get overridden by the "select first post" default. Fixed.
- **Projects page header stays pinned while scrolling (#347)**
  The header no longer scrolls away with the project list, matching the queue page.

### Removed

- **Queue import / export removed (#363)**
  The "Export / Import" buttons on the queue page and their endpoints are gone. Tasks and data jobs now live in a single ledger; to move between machines, migrate the `studio_data` directory instead (see the data-directory section in Settings).

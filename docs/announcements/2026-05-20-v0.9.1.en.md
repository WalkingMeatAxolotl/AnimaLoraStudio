Dependency fixes + version-consistency checks in the release pipeline

### Improved

- **bump_version.py now syncs package-lock.json, and a new verify-versions cross-file consistency check gates CI**
  The bump subcommand is extended to sync the version field in four places (studio/__init__.py, studio/web/package.json, and both the top level and `packages[""]` of studio/web/package-lock.json). A new verify-versions subcommand performs the consistency check and self-verifies after a bump; a new .github/workflows/version-check.yml runs validate and verify-versions on pull requests and on master / dev pushes, blocking version drift from entering history.

### Fixed

- **modelscope is now a required dependency, so the ModelScope download source no longer needs a manual install at runtime**
  The ModelScope path in services/model_downloader.py previously returned False when the package was missing and told you to run `pip install modelscope` by hand, since it wasn't listed in requirements.txt. This release promotes modelscope to a required dependency, so the ModelScope download source works out of the box after an install or self-update.

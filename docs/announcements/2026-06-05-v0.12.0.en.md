Preprocessing scope pushed down to the train version (ADR 0010) + Lion / Automagic optimizers + tag translation dictionary / autocomplete across the front end + WandB artifact uploads + InfoNoise algorithm alignment

### Added

- **Lion / Automagic optimizers + cosine_with_warmup scheduler (#214)**
  - **Lion** (Chen et al. 2023, arxiv 2302.06675): a sign-momentum optimizer whose state uses half the VRAM of AdamW; the formula is aligned line by line with Google's reference impl and lucidrains/lion-pytorch.
  - **Automagic**: AI Toolkit-style per-parameter adaptive lr + sign-agreement tracking + Adafactor factored 2nd moment + 8-bit lr_mask; the core is aligned 1:1 with tdrussell/diffusion-pipe `optimizers/automagic.py`. Adds a dependency on `optimum-quanto>=0.2.0`. The bf16 path uses Kahan compensated summation instead of stochastic rounding to avoid injecting ~scale/2 noise into the lr_mask every step; it deliberately doesn't register a `grad-accum hook`, sidestepping the upstream conflict that silently breaks `GradScaler.unscale_` / `clip_grad_norm_`.
  - **cosine_with_warmup**: a LambdaLR-based cosine schedule + linear warmup + eta_min floor; warmup uses step 0-indexing, consistent with the community convention across transformers / diffusers / PEFT / sd-scripts.
- **Tag translation dictionary + autocomplete across the front end (#225)**
  - **Default dictionary**: after first launch it pulls Physton/sd-webui-prompt-all-in-one-assets `danbooru.zh_CN.csv` in the background (~25k zh translations, ~3MB, **not in the repo**); on failure, Settings offers "restore default dictionary" to retry.
  - **Translated chip display**: TagEditor / TagsInput / the BulkActionBar dropdown / TagStatsPanel / the Overview tag distribution / the Regularization excluded tags — 6 places now show the Chinese translation consistently; the default on/off follows the i18n language on first run (`zh*` → on, otherwise off).
  - **Autocomplete input**: 6 input/textarea spots get completion — TagEditor chip + text mode, TagsInput (Settings + Tagging blacklist), TagStatsPanel inline rename, Regularization custom exclusions, and the Generate positive / negative prompt; Chinese input supports reverse lookup of the English tag.
  - **New tag-dictionary section in Settings**: current dictionary meta / upload a csv to replace / restore default / a "show translation" toggle.
- **WandB gains LoRA / training state artifact uploads + Settings / preset overrides (#146)**
  The WandB integration is more than metrics / sample logs:
  - Adds three kinds of artifact upload: the LoRA model, manual training state, and automatic training state.
  - Per-category retention: `all` keeps every version / `last` keeps only the latest (older versions are cleaned from the artifact collection automatically once a new version uploads successfully).
  - Defaults can be set in global Settings, and an empty preset field falls back to Settings automatically, so portable / cloud presets don't need to repeat the WandB config.
  - The upload path waits for completion via `run.log_artifact(...).wait()` before cleanup, avoiding operations on a draft artifact; large files get periodic progress logs.
- **Outputs gains bulk delete + unified copy-path / open-folder button wording and styling (#217, #224)**
  The "Outputs" panel shared by the Queue task detail page and the project Overview:
  - Bulk mode adds "delete selected (n)," which deletes by relative path after a second confirmation; if any file is missing, the whole batch is refused with a 404, leaving no half-deleted state.
  - Button wording: "Copy" → "Copy path," "Open" → "Open folder."
  - Button styling: "Open folder" changes from `btn-secondary` to `btn-ghost` to align with the adjacent copy / refresh buttons; "Delete selected" changes from a red `btn-danger` fill to `btn-ghost text-err`, keeping the danger cue in the text color only.
- **A sticky section-anchor nav in the right column of the training / preset pages (#224)**
  - Below the stats panel in the right column of the training page, and on the right of the preset page, a sticky section index is added; clicking smoothly scrolls to the matching SchemaForm group, and the current group highlights automatically as you scroll.
  - The main-area ratio on the training / preset pages changes from `1.5fr:1fr` to `3fr:1fr`, giving the field inputs more room to breathe.

### Changed

- **Preprocessing scope pushed down to the train version + a preprocessing phase added (ADR 0010) (#209-#213)**
  ADR 0010 pushes the working scope of preprocessing (dedup / upscale / crop) down from the project-level `download/` whole set to the training-version-level `train/`, switching the paths and server-side API across the board.

  - **Sidebar step order changes**: ① Download / ② Curate / ③ Preprocess (optional) / ④ Tag ... ⑦ Train. Preprocessing changes from "done on the whole set after download" to "done by each train version after curation," following the same pattern as regularization sets — "optional + skippable."
  - **The preprocess grid is the current version's train set**: it no longer shows the whole download set; different versions of the same project can dedup / upscale / crop independently.
  - **Path change**: the URL changes from `/projects/:pid/preprocess` to `/projects/:pid/v/:vid/preprocess`; old bookmarks auto-redirect to the current active version.
  - **Old projects migrate automatically**: on entering the new version, the `_v11` migration advances any version whose existing `train/{sub-folder}/` is non-empty from the `curating` phase to `preprocessing`; the old `projects/{id}/preprocess/manifest.json` is lazily converted to `versions/{label}/train/manifest.json` when a version is selected, transparently.
  - **Restore behavior changes**: when restoring an image, it's copied from `download/{origin}` back over `train/{name}`; if download no longer exists, the UI lists the "no origin" items so you can decide to drag in a replacement / keep / remove (the old version reported a missing error outright).
  - The old `/api/projects/{pid}/preprocess/*` endpoints / the old `preprocess_worker.run` project-scope dispatch are removed entirely (ADR 0010 PR-5). Third-party scripts calling the old URL need to switch to the version-scope path.
- **noise_enhancement description rewritten + pyramid default 0.5 + a mutual exclusion added with InfoNoise (#220)**
  After three rounds of empirical research, the descriptions and defaults of `noise_enhancement_type` / `noise_offset` / `pyramid_noise_*` were rewritten:
  - **`pyramid_noise_discount` default 0.35 → 0.5**: the old default made `noise_enhancement_type=pyramid` effectively a no-op (anima's `cur / cur.std()` normalization cancels a discount ≤0.4 back to baseline); pyramid noise only takes real effect from 0.5 up. yaml with an explicit value is untouched; only fallback-to-default values drift.
  - **Descriptions drop SD/DDPM-era lore**: `noise_offset` now states a 0-0.2 range, <0.05 equals off, >0.1 noticeably raises initial loss; `pyramid_noise_iters` states 0-6 controls frequency-band coverage; `pyramid_noise_discount` states 0.1-0.4 equals off, 0.5-0.7 noticeably changes low-frequency structure.
  - **`noise_enhancement_type != none` made schema-mutually-exclusive with InfoNoise**: this completes InfoNoise's 4 mutual-exclusion pairs (with `loss_weighting` / `loss_type=huber` / `timestep_schedule_shift` already landed in #216). InfoNoise's I-MMSE derivation assumes standard Gaussian noise — `noise_enhancement_type` alters the noise field, pulling the learned schedule away from its paper-optimal shape.
  - **Old configs don't auto-migrate**: an old yaml with `infonoise_enabled=true` plus any of the 4 mutually exclusive fields raises a `ValueError` outright at `TrainingConfig(...)` instantiation; the message names the conflicting field and offers an either/or choice: "(a) turn off InfoNoise and keep X; or (b) reset X to default and use InfoNoise."
- **Reg's "exclude train top tags" changed to a single column in descending frequency + hint removed (#218)**
  The Reg page's "exclude train top tags" card previously sorted tags into high / medium / low tiers by occurrence rate with a subtitle; user feedback was that the tiers didn't help actual selection. This version lays out the tag chips in a single column sorted by count descending, and the AI-mode meta line drops the "reverse-generate to push the concept back to base" hint, keeping only the "N excluded" count.

### Improved

- **lycoris LoRA forward defaults to bypass_mode, ~1.8x speedup (#196)**
  Issue #182: at the same model / config (rank=8 / batch=1 / Anima 1024), AnimaLoraStudio's 0.11 it/s was about 1.8x slower than sd-scripts' 0.19 it/s. The root cause: the default path of lycoris `LoConModule.forward` does a full-rank `F.linear` plus rebuilds the `(out, in)` ΔW matrix every step; across 280 patched layers the cumulative overhead is significant. This version defaults to lycoris's built-in `bypass_mode=True` (`org_forward(x) + lora_up(lora_down(x)) * scale`), matching the standard forward across the community (LoRA paper / sd-scripts / PEFT / diffusers); it's mathematically equivalent, and the speed returns to sd-scripts' level.
- **SettingsDrawer unmounts after closing + backdrop blur removed + local input state eliminates lag (#208)**
  - The drawer unmounts from the DOM once the close transition finishes, no longer maintaining a `backdrop-filter: blur(0px)` compositing layer for the whole viewport.
  - The overlay's `backdrop-blur-sm` changes to a plain semi-transparent color transition, eliminating scroll frame drops from real-time GPU blur on 2K/4K / integrated-GPU / scaled scenarios.
  - Adds a `SettingsInput` with local input state: typing only updates internal state, and `onBlur` / Enter sync to the parent's `draft` in one go, so the 4200-line SettingsPage no longer re-renders the whole page on every keystroke.
- **Training subprocess gets expandable_segments to reduce CUDA fragmentation and avoid low-VRAM OOM (#205)**
  At 8GB VRAM + LoKr full-matrix mode (27.5M params), PyTorch's default caching allocator accumulates "reserved but unallocated" memory, and OOM occurs once allocated + reserved approaches the VRAM ceiling. This version sets `os.environ.setdefault("PYTORCH_CUDA_ALLOC_CONF", "expandable_segments:True")` in the training subprocess before `import torch`, affecting only the training subprocess and not the shell environment; using `setdefault` doesn't override a value you've already set.

### Fixed

- **Several InfoNoise core-algorithm corrections + reg samples excluded by is_reg (#216, #223)**
  An end-to-end mock simulation (96 configs × 4 mmse shapes × 3 grad_accum × 2 baseline × 4 pivot strategies) exposed that the default InfoNoise crammed 99% of the probability mass into the σ_min quartile, so the adaptive sampler never actually took effect. This round fixes 4 algorithm-layer issues + 1 reg-set contamination:

  - **gate pivot `c` default 0.15** (the CIFAR value reported in the paper's §5): the old `above.argmax()`, because the generic 1/σ³ tail saturates `r_norm` to 1.0 at the low-σ end (the warning point in the paper's §B.6), always returned `c ≈ σ_min`, degenerating the gate `σ³/(σ³+c³)` into a near-identity. A new `infonoise_gate_pivot_c` field is an escape hatch: `0` follows the paper-faithful dynamic Eq 87, a positive value overrides explicitly.
  - **`r̂_k = mse / σ²`** (the paper's Appendix B.2 Eq 61, the log-σ entropy rate): the old `mse / σ³` came from the VE channel (Eq 59), inconsistent with summing over `Δlog σ`; the missing Jacobian is exactly what let the generic tail dominate normalization.
  - **`maybe_refresh` gated by `global_step`** (the optimizer step), no longer by `_internal_step` (the micro-batch count): `N_warm` and the default `total_steps × 20%` both assume optimizer steps, so `grad_accum=4` made warmup exit 4x early, going adaptive before the EMA had converged.
  - **`state_dict __version__ = 2`**: σ³ → σ² is a semantic break; resuming a v1 ckpt cold-starts and re-accumulates the EMA with a warning rather than raising, making long-run upgrades safe.
  - **Reg samples hard-excluded from InfoNoise records by the `is_reg` flag** (#223): previously a `loss_weight >= 0.99` threshold skipped reg samples, but the `reg_weight=1.0` boundary let reg still enter the records and contaminate the schedule. This now passes an `is_reg` flag through the dataset layer into the batch, and the loop masks with `~batch["is_reg"]`, decoupled from `loss_weight`. Reg samples still enter the gradient weighted by `reg_weight` as usual — they just don't enter InfoNoise schedule learning.
- **InfoNoise's 4 mutually exclusive field pairs get real-time front-end locking + tolerant disabling for old configs + a top banner (#221, #226)**
  InfoNoise's 4 schema mutual exclusions with `noise_enhancement_type` / `loss_weighting` / `loss_type=huber` / `timestep_schedule_shift` previously had only a backend validator, with no front-end disable_when. You weren't stopped while editing — you had to wait for the 600ms debounced auto-save to pop a `train.saveFailed` toast, with the form still dirty.

  - **Symmetric lock, non-default wins**: the 4 controlled fields get `disable_when=infonoise_enabled==true`; `infonoise_enabled` gets a reverse compound-OR `disable_when`, so when any conflicting field is non-default its checkbox grays out and locks. On a live switch, the other side resets to default automatically.
  - **Tolerant for old configs**: an old yaml with `infonoise=on` plus any conflicting field non-default, when entering the Train page, has the backend `_tolerant_validate` turn InfoNoise off automatically while keeping your original values, so you don't end up with all 4 fields grayed out and locked uneditable.
  - **Top banner**: a "Reset: infonoise_enabled" banner now also lights up on the initial config load (not only the fork-preset / save-as-preset paths), so you can immediately see which field was changed for you.
  - The "strict save" path (front end bypassing disable_when to PUT directly) still errors out, guarding against a silent value change.
- **VAE full-image decode OOM auto-falls back to tiled decode (#207)**
  When generating 1024×1024 reg images on 8GB VRAM, the VAE decode working peak can fill the remaining VRAM and OOM on the very first image. This version adds a try-full + `torch.cuda.OutOfMemoryError` fallback to `_tiled_decode` at the `VAEWrapper.decode` entry (tile=64 latent / 512 px, stride=48, cosine blend mask, fp32 accumulator to prevent bf16 precision loss).

  - **Zero cost on large VRAM**: it passes on the first try and never enters the tile fallback.
  - **Fallback on small VRAM**: a single tile's working peak is ~75 MB, each image is ~30% slower, but it completes.
  - **All 6 call sites benefit automatically**: reg_ai / generate CLI / generate XY / daemon generate / daemon XY / training sample preview share `sample_image` through the same entry.
- **cache_latents deduplicated by npz to avoid re-encoding the VAE for repeat folders (#199)**
  Kohya-style per-folder repeats (a `5_concept` prefix) expand the same image N-fold into `samples`; the old `_build_cache` walked `samples` directly to collect `to_encode`, so the same image was VAE-encoded N times, repeatedly overwriting the same npz — and `flip_augment` mode doubled that again. It now deduplicates by `npz_path`, so first-time cache build time scales with the unique image count; a second run (cache hit) behaves the same.
- **Reg AI priors support JSON captions + tags keep spaces instead of converting to underscores (#185)**
  AI-prior generation previously only read `.txt` captions and auto-converted spaces to underscores, so JSON-caption training sets couldn't be read for AI priors, and prompts became `brown_hair` rather than the space-separated tag format Anima recommends. This version:
  - reads all three caption formats — `.json / .txt / .caption` — keeping the space-separated tag form in the prompt;
  - writes `excluded_tags` back into the reg sidecar in sync (JSON's structured deleted field keeps the original form and `nl` natural language; TXT writes back after deletion);
  - matches excluded tags across spaces / underscores, so old booru / UI-style exclusions don't stop working;
  - full mode clears the old reg output first to avoid old captions mixing into training; incremental mode only adds new images.
- **CLI argparse compatible with py3.13+ + bare `%` in descriptions auto-escaped (#201)**
  Issue #170: on py3.13+, running `anima_train.py` hit `ValueError: invalid option name '--no-progress' for BooleanOptionalAction` (argparse tightened up and no longer allows a flag that itself starts with `--no-`). This version, when a schema bool field name starts with `no_`, uses a `store_true` / `store_false` pair instead of `BooleanOptionalAction`, with CLI behavior unchanged (`--no-progress` → True, `--progress` → False). Also auto-escapes bare `%` to `%%` in descriptions, so argparse's `format_help()` doesn't crash treating a description as a printf template.
- **Dataset uploads go through convert_to_png + same-stem files get an auto suffix (#222)**
  When `gelbooru.convert_to_png` is on, booru downloads normalize to `.png`, but dataset uploads / server-side path imports were raw byte passthrough — so downloading and uploading both could leave `1.png` and `1.jpg` (different images) in `download/` at once, and the later `{stem}.txt` caption pairing wouldn't catch them. This version routes uploads / zip entries / server-side path imports all through PIL decode + PNG re-encode + a same-stem collision suffix of `_1` / `_2` (no longer skipping your image on a name clash). The setting still reuses `gelbooru.convert_to_png` / `gelbooru.remove_alpha_channel` (the renaming move is left to a separate PR).

First-run onboarding upscaler-model download fix + clearer messages when Flash Attention fails to install

### Improved

- **Clearer error messages when Flash Attention fails to install, with an early prompt to reinstall if you're on CPU PyTorch (#275)**
  - Flash Attention install failures are usually caused by having the CPU build of PyTorch (flash_attn requires the CUDA build), but previously all you saw was "Internal Server Error" with a trace_id and no hint at the cause.
  - As of this release, when the CPU build of PyTorch is detected in the Settings → Training → Flash Attention section and in first-run onboarding, you get a clear "reinstall the CUDA build in the PyTorch section first" message and the flash_attn install button is disabled, avoiding a pointless download.
  - Other unexpected errors (GitHub Releases unreachable, an incompatible wheel, and so on) now report their specific type too, so the whole section no longer collapses into "Failed to load" because of a backend surprise.

### Fixed

- **Fixed an error downloading upscaler models from first-run onboarding (#275)**
  - Previously, clicking "Retry" on an upscaler-model entry always got stuck on `unknown model_id 'upscalers'`, so anyone wanting to install upscaler models (4x-AnimeSharp and the like) from onboarding had to detour through Settings; one-click downloads work again as of this release.

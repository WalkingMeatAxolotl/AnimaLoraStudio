修复 Block 交换开启时训练结果不正确的问题

### 修复

- **开启 Block 交换后训练结果不再受影响（#466）**
  v0.21.0 中，训练时把「Block 交换」设为大于 0 会让梯度计算出错，训练全程没有任何报错或提示，产出的 LoRA 质量下降。配合 Prodigy 系优化器（PPSF）时表现为学习率估计值突然跳升、训练中断；配合 AdamW 等固定学习率优化器时则看不出异常。**如果你在 v0.21.0 里开着 Block 交换训练过 LoRA，建议升级后重新训练。** 未使用该功能（保持默认 0）的训练不受影响，测试出图的 Block 交换也不受影响。

Unified Comfy-parity sampling pipeline + T-LoRA / OrthoLoRA adapters + resume training on failed/canceled tasks + Generate-page history backfill + first-run onboarding + project-page search/archive + custom studio_data location

### Added

- **Test, training-sample, and RegAI sampling unified on Comfy-parity runtime v2 (#256)**
  The three sampling paths — test image generation, training-sample previews, and RegAI prior generation — are merged into one implementation, all aligned with ComfyUI KSampler behavior; the old parity toggle is gone.

  - Added the `dpmpp_3m_sde` sampler and the `sgm_uniform` scheduler, matched parameter-for-parameter to their ComfyUI namesakes.
  - Training sampling gains a "Comfy encoding" toggle (on by default): the full caption text is sent to Qwen3 + T5 as-is, with booru bracket notation kept literal; turning it off uses the old per-tag encoding, which is handy for A/B-ing old vs. new and for resuming a checkpoint trained under the old encoding.
  - Negative prompts now match ComfyUI: leave it empty and it's an empty string — no implicit default negative prompt. If an old preset has a sampler / scheduler value that's no longer supported (like `euler`), it's folded back to the default.
  - Settings → Test gains a VAE decode precision dropdown (bf16 / fp32), defaulting to bf16; on modern GPUs the default path no longer does unnecessary model moves.
  - Occasional attention NaNs during generation now drop to a fallback implementation and reset automatically after the current image finishes, instead of permanently downgrading the whole process.
- **T-LoRA + OrthoLoRA adapters + Automagic v2 experimental path (#215, #258)**
  The training config's `lora_type` gains two new variants aimed at small-dataset character / subject customization.

  - **T-LoRA** (arXiv 2507.05964): adapts rank per timestep — capping rank in high-noise steps and using full rank in low-noise steps — to ease overfitting on small datasets.
  - **OrthoLoRA**: layers an orthogonal rotation + scaling on top of the LoRA base; you can pick `ortho` on its own or stack it with T-LoRA (the `tlora_use_ortho` toggle, on by default — the full recipe from the T-LoRA paper is the ortho variant). **It's automatically distilled into a standard LoRA on save**, so inference engines like ComfyUI load it at zero cost.
  - Every `lora_type` option's description was rewritten as "what the parameter does + when to use it", instead of just listing algorithm names.
  - Experimental Automagic v2 (per-param scalar LR + fused backward to save VRAM): not exposed in the UI — it only appears if you manually set `enable_automagic_v2` to true in `studio_data/secrets.json`. The CLI / yaml path is unaffected.
- **Failed / canceled training tasks can resume from an epoch backup point (#266)**
  Since the last release, an automatic backup point has been written at the end of every epoch, but the "Resume training" button was previously only available for paused tasks — even though the backup points for failed (crashed / shut down) and canceled tasks are usually intact. This release opens it up.

  - The Queue list and Queue detail page add a "Resume training" button on failed / canceled rows, alongside the existing "Retry (from scratch)". Done tasks can still only be retried.
  - Recovery survives a reboot: the backup point's location is recorded in the database, so it isn't lost even if the process or machine dies. Existing tasks from the 0.12 era are backfilled into the new mechanism automatically.
  - Automatic backup points now land in the task archive directory (`studio_data/tasks/<id>/state/`), alongside sample images and run logs for easy archiving as a unit. The location of training state you save yourself periodically is unchanged.
  - Writes are now atomic, so a power loss won't leave a half-written backup file; after a successful recovery the backup point is no longer deleted, avoiding a window with no recovery point between resuming and the next epoch.
- **Generate-page history click restores parameters + saved PNGs carry a parameter snapshot (#239, #244, #245)**
  The Generate page's history strip is upgraded from "just swap the main image" to "restore the parameters too"; a new Settings → Test → auto-save toggle saves images straight to `studio_data/test/<date>/` when enabled.

  - Clicking in the history strip: in xy mode it restores both the xy axes and the LoRA list; in single mode it restores the LoRAs and the image count. A misclick gets a 5-second undo prompt.
  - Saved PNGs embed a full parameter snapshot — drag one back into Studio and it reads, drag it into ComfyUI / WebUI and the prompt comes out too (single mode also writes the a1111-compatible format).
  - Across sessions / cache clears / device changes: Studio scans the save directory at startup to repopulate the history strip, no longer relying on browser local storage.
  - The × button in the top-left of each history thumbnail is removed (too easy to hit by accident); individual deletion now goes through "Clear" / "Clear stale" at the top.
- **First-run onboarding modal — a basic-capabilities checklist with one-click install (#238)**
  New users no longer have to hit a wall during training / tagging and then go back to Settings. The onboarding pops up right after you pick a language in the language modal.

  - Four checklist items: the base-model suite (the Anima base model + VAE + Qwen3 + T5), tagging (WD14 + ONNX Runtime), training acceleration (flash-attn / xformers), and image enhancement. Anything not installed shows "Not installed (X/N)"; check it → one click to start downloading.
  - Chinese defaults to ModelScope and English to HuggingFace, written into the download source automatically. During install it shows live download speed / ETA / current file.
  - Settings → System adds a "Re-run first-run setup" button to bring it back anytime.
- **studio_data directory location is customizable + copy-based migration (#263, #267)**
  If your install lives on the system drive, you can move the ever-growing project data to a larger drive. Settings → System → Storage location: pick a target directory and a confirmation modal appears (showing the number of files to copy / total size / top-level breakdown); once migration starts, a progress bar advances in real time.

  - **Copy only, never delete**: data at the old location is left exactly as is; if migration fails, the half-written target directory is cleaned up and it's as if nothing happened.
  - The database is backed up with an online snapshot, so it isn't captured mid-write during migration.
  - A `studio_data/` subdirectory is created under the directory you chose and moved into (it won't scatter contents across the parent directory you picked).
  - If the pointer file is corrupted or the drive isn't mounted, it falls back to the default location automatically, and old data isn't lost.
  - Also fixed: newly created version directories no longer generate an empty `samples/` folder (sample images actually land in the task archive directory).
- **Project page gains search / filter / sort + soft-delete archiving + "Preparing" shows the current stage (#265)**
  A browsing overhaul for when you have a lot of projects: find projects, collapse inactive ones, and see which step each project is stuck at.

  - The header "Filter ▼" toggle takes no space when collapsed; expanded, it's a single row: search (title / slug / notes, case-insensitive) + a status dropdown + a sort dropdown.
  - The × in a card's top-right corner changes from hard-delete to **archive**: the directory / versions / tasks are all kept and can be restored anytime. The header's "Archived (n)" switches to the archive view, where archived cards are semi-transparent and offer Restore and permanent-delete buttons. Archiving doesn't update "last active time", so a restored project returns to its original spot rather than jumping to the front.
  - The "Preparing" status badge now shows the current stage (e.g. "Preparing · Tagging"), covering all six stages (Download / Filter / Preprocess / Tagging / Regularization set / Training).
- **Test-generation model auto-unloads when idle to free VRAM (#232)**
  - Solves the common "switched to another page and forgot to clean up, then ran out of VRAM next time I started training" scenario: once the Generate page has loaded the model, it auto-unloads to free VRAM after sitting idle past a set time.
  - Defaults to 10 minutes; adjustable in Settings → Test, set 0 to disable. Changing the value takes effect immediately while running.
- **Tagger gains an "existing caption" policy: overwrite / skip / append (#237)**
  - The tagging page's top bar adds an "Existing caption" dropdown next to "Format": **Overwrite** (default, the original behavior) / **Skip** (images that already have a caption are left as-is) / **Append to end** (merged and deduplicated by tag, written back in the original format).
  - All three taggers — WD14 / CLTagger / LLM — support it. In append mode the trigger word isn't added twice.
  - This is a per-task option, not in global Settings.

### Changed

- **Task logs across the whole app are unified into a three-state drawer (Download / Preprocess / Tagging / Regularization set — six pages in all) (#253)**
  Each page's task log used to be five different implementations behaving differently; this release unifies them into a three-state drawer docked at the bottom.

  - With no task it's hidden entirely; with a task it shows a one-line collapsed bar by default (status badge + last log line + elapsed time), so you can see progress moving without expanding.
  - It auto-expands when a task enters the running state (including the replay case where you return to the page and find the task still running); it doesn't auto-collapse when the task finishes — collapse it or switch pages yourself.
  - Also fixed: an old historical booru task on the regularization-set page no longer permanently covers the AI-prior log that's currently running; preprocess upscale / crop tasks get log replay; dedup-scan failures now go to the log (previously only a toast).
- **ONNX Runtime moves to manual install in Settings + new DirectML support (#236)**
  - Studio no longer auto-runs pip install onnxruntime (~150MB / several minutes) at startup; when it's not installed it's silently skipped — faster startup, and users who don't need WD14 / CLTagger aren't blocked.
  - New DirectML backend: it goes through DX12 and **doesn't depend on CUDA / cuDNN / cuBLAS**, so any DX12 GPU (NVIDIA / AMD / Intel, Windows-only) can run it. Fixes the issue where the original onnxruntime-gpu silently fell back to CPU on an RTX 5090 (issue #231).
  - Settings → ONNX Runtime has three buttons: DirectML / CUDA / CPU; auto-detection routes by platform (Windows + GPU → DirectML / Linux + GPU → CUDA / no GPU → CPU).
  - When onnx isn't installed, the tagging page shows a "Go install ONNX Runtime →" button that jumps straight to the matching Settings section.
- **Tag-translation dictionary's default source switched to ffdkj (updated daily, 310k entries) (#268)**
  - The previous default, Physton, has been unmaintained for 2.5 years (last updated 2023-11, 99k entries). The new ffdkj source updates daily and covers 311k entries (every tag with post_count ≥ 10, Gemini-translated + human-proofread).
  - The top 200k entries by popularity are loaded; what's cut is the lowest-reference long tail.
  - The flow for uploading your own custom dictionary is unchanged; Settings "Restore default dictionary" re-downloads from the new source.

### Improved

- **Tag autocomplete sorting changed to popularity-first + compact matching (#248)**
  - It used to sort by tag-name length, so typing "red" put obscure short tags ahead of "red eyes". Now it sorts by popularity (post_count descending).
  - Added compact matching: it also matches after stripping spaces / underscores, so "redey" matches "red eyes".
- **Changelog modal gains previous/next version navigation + a dedicated Settings entry point (#269)**
  - The changelog detail modal could previously only show the "currently displayed version", so after a hotfix the content of an older feature would disappear from the UI. The modal gains previous/next buttons + a `(n / N)` position, and the keyboard ← / → switch too.
  - The version card in Settings → System gets a dedicated "📜 Changelog" button in its top-right corner that opens the modal and jumps to the latest version.
- **Generate page gains sampler / scheduler dropdowns + auto-growing prompt inputs (#262)**
  - The Generate page's "Sampling parameters" card fills in the sampler and scheduler dropdowns (`er_sde` / `dpmpp_3m_sde`, `simple` / `sgm_uniform`), and your choice is remembered locally.
  - Enum labels drop underscores and are capitalized correctly (e.g. `dpmpp_3m_sde` → `DPM++ 3M SDE`).
  - The positive and negative prompt inputs are at least 5 lines tall and grow with content, so you no longer need to drag or scroll.
- **VAE cache batching defaults to follow the training batch size + more memory-efficient weight loading (#254)**
  - `vae_cache_batch_size` now defaults to `0` = follow the training batch size (matching kohya GUI's `vae_batch_size` semantics). Setting it to 1 manually still keeps the conservative path.
  - Fixes a possible OOM when first building the VAE cache on a low-VRAM card with a 3D VAE at full resolution.
  - Saves one memory copy during weight loading.
- **Half-loaded thumbnail loads are canceled on route changes to avoid starving data requests (#264)**

### Fixed

- **LoKr layer-wise rank inference strength now matches training (#241)**
  Layer-wise-rank LoKr previously loaded at a strength inconsistent with training in inference engines like ComfyUI. The metadata saved in this release writes out each layer's actual rank and alpha, and the inference side reads them per layer to scale, matching the training forward path.
- **Booru downloads distinguish 429 / 503 + no more permanent slowdown + log dedup (#234)**
  - When the gelbooru server hit an occasional 503, the log mislabeled it as "429 backoff, waiting 59.5s", steering troubleshooting the wrong way, and a single 503 would permanently halve the download rate (unrecoverable).
  - The two are now distinguished: 429 ("you're too fast") → back off 60 seconds + halve the rate; 503 ("I'm down") → back off 15 seconds, **leaving the rate alone**.
  - The API and CDN back off independently, so an API search isn't dragged down during a CDN storm. Consecutive 503s within the same backoff window now log just one line instead of flooding.
- **Studio showed a blank screen on some Windows machines (now actively registers .js / .css MIME types) (#233)**
  On some Windows machines, because the system registry had been altered by other software, the browser got incorrect MIME types for `.js` / `.css`, causing Studio to load to a blank screen. The backend now registers the correct types to avoid reading from the registry.
- **Tag chip-mode input: Backspace no longer accidentally deletes a confirmed tag (#247)**
- **Preprocess quality-candidate review panel layout — responsive multi-column cards + full height (#252)**
- **Minor UI fixes: log replay / tagging skip pre-filter / crop leftovers / dedup panel (#249)**
  - Restore live logs on the RegAI prior-generation page and the tagging page after switching pages / refreshing.
  - When tagging with "Existing caption = Skip", images that already have a caption are filtered out up front, avoiding spinning up the tagger for nothing.
  - After a training-scope crop finishes, old source images and sidecars that are no longer part of the output are deleted, fixing the issue where the original + cropped images both lingered after a bundle / version copy.
  - The dedup quality-review panel switches display mode (grouping / quality) to keep panels from crowding each other on narrow screens; expanding the tagging advanced options no longer squeezes out the log panel.
  - Bundle import/export preserves `version_label` and `preset_name`, restoring them on import instead of fixing them to v1.
  - Checkboxes / radios use a unified theme accent color.

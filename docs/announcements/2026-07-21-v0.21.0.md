Block 交换：12 GB 显卡训 Krea 2、8 GB 显卡出图 · 测试出图显存与稳定性 · 落盘图片带 Civitai 兼容信息 · 训练配置所见即所得

### 新增

- **Block 交换：12 GB 显卡训 Krea 2、8 GB 显卡出图（#463）**
  把模型靠后的若干层放在内存里，算到哪一层再搬进显存，用时间换显存。训练和测试出图都能用：训练配置的「系统与性能」和 **设置 → 测试 → 显存策略** 里都有「Block 交换」，默认 0（不开启）。怎么用、填多少、小显存显卡要注意什么，见公告「Krea 2 小显存训练指南」。

  - RTX 5090 / fp8 底模 / 1024² 实测：换出全部 28 层时训练整卡约 10 GB、出图约 6.3 GB，速度慢约 4%
  - 只改变权重放在哪，不改任何训练数值；出图结果与不开启时一致
  - 换出的层常驻内存，需要相应的空闲内存（28 层约 11 GB），内存不够时会在开始前告诉你
- **落盘的测试图带 Civitai / A1111 兼容信息（#456）**
  把图拖进 Civitai、A1111 或 ComfyUI 时可以直接读到底模、VAE、LoRA 的名字与 hash。图里只写文件名和 hash，你机器上的绝对路径不会进图。

  - XY 合成图仍只带 Studio 的参数快照，完整信息写在每个格子的原图上
- **模型路径可以从已下载的模型里选（#461）**
  训练配置的底模 / 文本编码器 / VAE / tokenizer 四个路径字段新增「选择模型」，列出磁盘上已就绪的模型，选中即填路径。没有候选时提示去设置页下载，手填和「浏览」照旧可用。
- **标签编辑页与训练集筛选页可以拖分栏（#457）**
  拖动栏之间的分隔条调整宽度，比例会记住；分隔条选中后方向键也能微调。窄屏下仍是上下堆叠。

### 变更

- **训练配置所见即所得（#458, #460）**
  你保存的版本配置不再被系统改写——点「开始训练」入队时配置文件一个字节都不动。随之有两个行为变化：

  - 在全局设置里换了底模，已有版本**不再自动跟随**。字段旁标注「默认来自全局设置」，与全局当前值不一致时出现「恢复默认」链接，点一下即可对齐
  - 训练配置里的「触发词」字段退役，采样 prompt 不再自动带上触发词。如果之前依赖它，把触发词直接写进采样 prompt 即可；打标页的触发词不受影响
- **AI 生成正则集默认走增量模式（#453）**
  切换标签后不再自动回到「全量」，避免误点重新生成把已生成的图清空。
- **各步骤页的图片顺序统一（#459）**
  预处理 / 裁剪 / 涂抹 / 标签编辑 / 数据集 / 正则集现在都按 booru id 数值排序，与训练集筛选页一致；此前 `10.jpg` 会排在 `9.jpg` 前面，图一多就跟别的页面对不上。

### 改进

- **测试出图的显存峰值下降、更不容易中途失败（#455）**
  VAE 推迟到解码阶段才占显存；遇到未缓存的 prompt 需要重新加载文本编码器时，按显存策略先让 DiT 让位。32 GB 实测第二轮新 prompt 的全流程峰值从 21.2 GB 降到 17.7 GB。

  - 显存瞬时不足导致的单张失败会用相同 seed 自动重试一次，不再中断整批出图
- **新增 LoRA merge 精度选项（#455）**
  fp8 底模挂 LoRA 是把 LoRA 合并进权重的，**设置 → 测试** 新增这一步的计算精度：fp32（默认）与 ComfyUI 一致，bf16 计算量减半、切换 LoRA 更快，代价是结果可能有轻微数值差异。不影响采样阶段的显存。

### 修复

- **自定义的「保存的模型名称」不再在开始训练时被改回默认（#458）**
  此前保存后 YAML 预览正常，一点开始训练就变回 `{项目}_{版本}`。
- **接续训练的恢复点不再在入队时被清空（#460）**
  v0.20.0 起，选好 resume LoRA / 训练状态后点开始训练会被静默清掉，实际从头开始训。
- **多 prompt 出图时 PNG 里记录的 prompt 与 LoRA 不再写错（#456）**
  此前一批多个 prompt 时每张图都写第一个 prompt；XY 矩阵里未被轴引用的 LoRA 也会被写进图。

Preprocessing pipeline + InfoNoise training + a major Generate/Preset UX overhaul (ADR 0004)

### Added

- **New "Preprocess" step (image upscaling) with live progress (#69)**
  A new "② Preprocess" step is inserted between ① Download and ② Filter in the pipeline (the later steps shift down), letting you upscale all of your downloaded images before moving on to filtering. Existing projects can skip it (marked "(optional)" in the sidebar).

  - **Multiple upscaler presets**: the Settings → Preprocess tab ships with ESRGAN / Real-ESRGAN presets plus custom-repo downloads, from either HuggingFace or ModelScope
  - **Smart pipeline**: a target-area threshold lets large images skip the upscaler model and resize directly, saving time
  - **Automatic GPU fp16**: the worker logs device diagnostics at startup, so you can see whether it's running on CUDA or CPU and whether fp16 is active
  - **Progress over SSE**: per-image completion events stream to the frontend in real time (no more polling); while a job runs, files are polled every 3s to refresh the grid, progress, and disk usage
  - **Pre-generated stage thumbnails**: 256 and 768 sizes are generated ahead of time for the grid, so switching filters stays smooth
- **New InfoNoise adaptive timestep sampler (#63, #66)**
  Built on the I-MMSE identity (`dH/dσ = mmse/σ³`): it tracks per-bin denoising MSE with a FIFO and EMA, then builds an inverse-CDF sampler that concentrates sampling in the most informative noise windows; during warmup it falls back to the logit-normal baseline.

  - **No impact by default**: seven `infonoise_*` fields, all off by default; existing training behavior is unchanged
  - **Automatic N_warm**: `infonoise_N_warm = 0` computes `max(200, total_steps × 1/5)` automatically — short runs are no longer stuck on a hardcoded 5000 steps
  - **Plugin registry**: a new `runtime/training/timestep_samplers/` subpackage where the baseline and infonoise samplers share one protocol; adding future samplers won't touch phases or the loop (following the ADR 0003 pattern)
  - **EMA formula follows the paper's Algorithm 1**: `mse ← (1-β)·mse + β·ℓ̄`, with a docstring warning up top not to flip it on intuition (PR #65 flipped it by mistake → PR #66 restored it and added 16 regression tests to codify the formula)
  - **Cold-start trip wire**: warns once if the CDF stays not-ready for too long, and exposes `status()` for external probing
- **Simple / Advanced mode toggle on the Train and Presets pages (#63, #66)**
  A Simple / Advanced toggle now sits in the top-right of the Train and Presets pages. Simple mode hides 35 advanced fields by default (dropout, scheduler fine-tuning, PPSF knobs, noise-schedule details, and more) so new users aren't overwhelmed.

  - The state lives in a `useAdvancedMode` hook backed by `localStorage`, updating live within a tab; storage events sync it across tabs, so two open tabs won't drift
  - The schema is regrouped: a new `noise_schedule` group (noise offset, pyramid noise, timestep sampling, InfoNoise, loss weighting); `kv_trim`, `mixed_precision`, `attention_backend`, and `num_workers` move into the `system` group
  - Field descriptions adapt to context: when InfoNoise is on, `timestep_sampling` is disabled with an explanatory note, and `timestep_shift` shows warm-up context
- **studio CLI gains --torch / --fe-port flags and a default-subcommand fix (#63)**
  - **`--torch <tag>`** (`cu128` / `cu126` / `cu124` / `cu118` / `cpu`): forces a specific CUDA torch wheel, for rented CPU-only machines where the GPU torch build can't be auto-detected. Ctrl+C skips it; `--skip-pending` also skips the pending install on restart
  - **`--fe-port`** (the `dev` subcommand only): the Vite dev server port, default 5173
  - **Default-subcommand fix**: `studio.sh --port 6006` used to fail with `invalid choice: '6006'`; an unknown positional now correctly falls back to `run`
  - pip installs stream their output, and Ctrl+C gracefully interrupts a pending torch install

### Changed

- **The preprocess UI is consolidated into a single grid, so you no longer have to distinguish "downloaded" from "preprocessed" images (ADR 0004) (#74)**
  ADR 0004: the two-grid layout (pending / processed) is flattened into a single grid with status badges. From your point of view there's only one copy of each image, and processing state is just a property of it.

  - **Single source of truth for state**: `projects/{id}/preprocess/manifest.json` replaces the per-image `*.preprocess.json` sidecars
  - **One ImagesPanel**: a summary header ("N total · X unprocessed · Y processed") plus filter chips (All / Unprocessed / Processed) and per-card status badges (✓ upscaled / ⊘ unprocessed)
  - **A "⟲ Revert" button** replaces "🗑 Delete": reverting deletes the preprocessed copy and returns the image to unprocessed
  - **Seam collapsed**: the `left_source` field and the dual-URL branch are gone; the frontend always uses `projectThumbUrl` and is unaware of any download / preprocess distinction
  - **Zero-deletion migration**: the first time you open each project, if there's no manifest but old sidecars exist, they're aggregated into a new manifest.json — the old sidecars are kept, not deleted (a defensive fallback)
- **LoRA picker rewrite + unified click behavior across three image grids (#77)**
  PR #70 showed as Merged on GitHub because of the squash-merge but never actually landed on dev (revived via cherry-pick in #77 — see the PR description).

  - **LoRA picker rewrite**: the flat search becomes a three-stage project → version → ckpt set of dropdowns plus a chip list; single and multi modes are separated, and the weight slider shows only when needed
  - **Unified click behavior across the Download, Preprocess, and Reg image grids**: a click opens the lightbox preview; checkbox / Shift / Ctrl / ⌘ + click selects multiple (matching the existing Curation / TagEdit convention)
  - **PromptFromDatasetPicker**: a controlled single-select that stays visible, with a read-only tags area and a `datasetSuffix` appended to the end of the prompt
  - **XY matrix**: the lora_ckpt axis becomes a multi-select chip ("Add N") and `lora_scale` becomes a global axis (no longer tied to a specific LoRA)
- **Preset import/export now uses yaml files (replacing JSON) (#79)**
  The backend already stored presets as yaml, but the frontend exported JSON plus a hand-written 4-line mini-YAML parser (single-line scalars only, silently dropping list / nested / quoted fields such as `sample_prompts` / `timestep_samplers`). This PR switches the transport model to end-to-end yaml files.

  - New `GET /api/presets/{name}/download`: a FileResponse streams the on-disk yaml straight through, so the exported file is byte-for-byte identical to `studio_data/presets/{name}.yaml`
  - New `POST /api/presets/import`: a multipart `UploadFile` that the backend runs through `yaml.safe_load` and pydantic validation, returning `{config, suggested_name}` to the frontend draftSeed flow (nothing is written to disk until you rename and confirm)
  - **Old `.json` exports still import**: yaml.safe_load is a superset of JSON, so there's no compatibility burden
  - A closed-loop workflow: you can edit `studio_data/presets/{name}.yaml` directly and import it back, and on schema-validation failure the error message comes from a single source — the backend `PresetError`
  - Button label "Export JSON" → "Export YAML"

### Improved

- **Tag editing adds "current filter / current list" options to its bulk scope (#71)**
  After filtering, the bulk scope used to offer only "current selection" and "all images": the former meant manually selecting everything, and the latter would change hidden images outside the filter. "The batch I'm currently looking at" is now an explicit scope.

  - All three scopes carry a count: "Selected (K)" / "Current filter (N)" / "All images (M)"
  - Bulk add / remove / replace / dedupe all support switching scope
  - When the filter returns no results, the bulk buttons are disabled to prevent acting on an empty set
- **A set of fullscreen interaction improvements across Generate / Queue / XY (#64, #68)**
  - **XY fullscreen cell navigation**: arrow keys / PgUp / PgDn move between cells; edge hints dynamically show which directions are available (no longer falsely promising all four); focus isn't stolen while you're in an input or textarea
  - **PreviewCompare side-by-side**: in fullscreen, ← / → switch between A and B (previously only ESC worked)
  - **InlineLoraPicker header**: split into two rows (search / actions, plus project / version dropdowns); the dropdown row is hidden when there's one project or fewer
  - **PromptFromDatasetPicker persistence**: the dataset prompt you picked is remembered across refreshes
  - **`useLocalStorageState` hook**: generic JSON serialization, cross-tab sync via storage events, and SSR safety; old `anima.` localStorage keys migrate automatically to `studio:`
  - **Shared ConfigSkeleton component**: aligns the loading state of Train and Presets visually
  - Generate sample responses now send `Cache-Control: no-store`, forcing a fresh fetch every time
- **wandb training logs: epoch metrics reorganized + log_samples toggle restored (#63, #65, #66)**
  - Dropped the per-step `train/epoch` (the step axis already covers it), added `train/loss_epoch` (the epoch mean), and `train/epoch` now logs only once at each epoch boundary
  - **`log_samples` toggle restored** (PR #63 removed it by mistake → PR #65 fixed it): defaults to True (enabling wandb uploads sample images, keeping things convenient); for NSFW or private datasets you can turn it off in Settings and upload metrics only
  - The `wandb` dependency is promoted from optional to a hard `requirements.txt` dependency

### Removed

- **Removed the fake "Recycle Bin" and the queue's "Clear completed" button (#78)**
  Both buttons shared one problem: the UI implied "recoverable / frees space" but didn't deliver.

  - **Recycle Bin**: deleting a project or version used to move it to `studio_data/_trash/`, but there was no restore UI and no automatic cleanup — so from the UI it behaved like a hard delete while silently accumulating orphaned directories on disk. It now calls `rmtree` directly; the delete confirmation stresses "this can't be undone" with a red danger tone
  - **The queue's "Clear completed" button**: it only deleted the done rows from the `tasks` table (kilobytes), leaving `logs/{id}.log` (megabytes) and output (per-version, unrelated to the task row) untouched, and it missed `failed` / `canceled` states. The button is gone (proper log / output cleanup will come as its own feature)
  - Also marked "② Preprocess" and "⑥ Regularization set" as "(optional)" in the Sidebar / Stepper

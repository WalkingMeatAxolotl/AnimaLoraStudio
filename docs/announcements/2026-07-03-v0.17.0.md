队列全面改版 · 出图多任务排队 · 训练定时开始 · 显存调度统一准入 · 打标落盘格式收敛

### 新增

- **队列页全面改版：分区、全类型可见、搜索 / 过滤 / 分页（#343, #348, #359, #361, #362, #363）**
  队列不再只显示训练任务：训练 / 正则集构建 / 测试出图 / 评估出图统一进一个队列，按「计划任务 / 进行中 / 等待入队 / 历史」分区显示，历史支持分页。

  - 顶部可搜索（任务名 / 配置名 / 项目名）、按类型和状态过滤；筛选条件会记住，切走再回来不丢。默认只看训练，可切「全部」。
  - 详情页按任务类型显示对应内容，并提供「查看结果」跳转（出图任务直达测试页对应结果，正则任务直达正则集页）。
  - 新增「数据作业」标签页：下载 / 预处理 / 打标 / 评估指标等作业集中可见，行内展开看参数、错误与实时日志，可取消、可跳回原步骤页。
  - 所有任务与数据作业现在共用一套编号，跳转与引用不再有两套 ID。
- **出图多任务排队：连点连出、统一时间线、全阶段进度（#355, #348）**
  测试页点「开始生成」不再互相打断：任务依次排队执行，右栏变成统一时间线（排队中 / 生成中 / 已完成），可单条取消或一键清空队列。

  - 「开始生成」旁新增批量数：一次入队 N 个任务，seed 自动递增。
  - 进度条覆盖模型加载 / 文本编码 / 采样 / VAE 解码全阶段，不再长时间卡在 0% 或 100%。
  - 出图任务的日志现在也能在队列详情页实时查看，并可从详情页直达对应出图结果回看。
- **训练可定时开始（#356）**
  训练入队时可设定具体时间（如凌晨 3:00）或延迟 N 小时，到点自动进入排队；队列页新增「计划任务」段，显示计划时间与倒计时，可随时「立即开始」或取消。
### 变更

- **显存调度统一为资源档位准入（#360, #362, #363）**
  什么任务能同时用显卡，现在由后端按三档统一把关：独占（训练 / 正则 AI 先验 / 测试出图 / 评估出图）全系统同时只跑一个；轻量（打标 / 超分 / 评估指标）可与独占任务并行；下载不受限。修复了训练时仍可能同时出图等三个显存准入漏洞。

  - **训练时打标 / 超分 / 评估指标现在默认照常并行**（此前默认推迟）。可在设置里关闭；旧开关 `allow_gpu_during_train` 废弃，新开关为 `light_tasks_during_train`，旧值不迁移。
  - 独占类任务严格按入队顺序执行，不再有隐式插队——所见队列顺序即执行顺序。
  - 训练时测试页「开始生成」不再置灰：现在可以直接提交，排队等训练完成后自动开始。
- **本地打标落盘格式跟着产物走，不再手选（#353）**
  打标页的「输出格式」选择器移除：本地打标（wd14 / CLTagger）与 LLM 文本模式一律写 `.txt`（触发词在首位），LLM 分类 JSON 模式写 `.json`；图片已有 `.json` 的保持 `.json` 更新。

  - `.txt` 标注现在支持 `tag_dropout`（此前只对 JSON 生效）；`keep_tokens` 前缀照常免 shuffle 免 dropout，与 kohya 语义一致。
  - 已有的扁平 JSON 数据集不受影响，训练端永久保留读取兼容。

### 修复

- **本地打标 + JSON 输出的数据集不再以空标注参训（#352）**
  本地打标器写出的扁平 JSON 此前会被训练端误判格式、解析失败后静默以空 caption（连触发词都没有）参与训练。现在读取正确，且开训前会预检所有 JSON 标注，有解析失败的直接列出文件、拒绝开训。
- **`lora_reg_dims` 分层 rank 此前对 `algo=lora` 静默失效（#350）**
  自 v0.10.0 起，`algo=lora` + `lora_reg_dims` 的训练分层 rank 一直被静默忽略（实际按全局 rank 训练，无任何报错；LoKr 不受影响）。修复后同配置重训会真正按分层 rank 构建网络。

  - 注意：旧的此类 checkpoint（统一 rank）在新版本 resume / 继续训练会因形状不匹配报错——去掉 `lora_reg_dims` 配置或重新训练即可。自检：受影响 run 的 `run.log` 里只有 `[lora_reg_dims] skipped N modules` 而没有 `applied rank overrides`。
- **训练配置页「采样 prompts」现在能正常换行（#364）**
  多 prompt 轮换字段（每行一条）此前敲回车会被立即吃掉，实际没法输入多行。已修复；同页三个采样 prompt 输入框改为随内容自动撑高。
- **公告栏打开瞬间点别的公告不再被跳回第一篇（#354）**
  打开公告栏后立刻点另一篇时，选中可能被「默认选中第一篇」覆盖回去。已修复。
- **项目页顶栏滚动时保持固定（#347）**
  项目列表往下滚时页面顶栏不再跟着滚走，与队列页行为一致。

### 删除

- **队列导入 / 导出功能移除（#363）**
  队列页的「导出 / 导入」按钮及对应接口移除。任务与数据作业已统一进单一台账，跨机迁移场景请直接迁移 `studio_data`（见设置页的数据目录说明）。

Hotfix — a double failure in the webui self-update path for v0.10.2 users

### Fixed

- **v0.10.2 users always saw "you're up to date" when checking for updates and couldn't see v0.11.0+ (#198)**
  Users on v0.10.2 who clicked Settings → System → Check for updates saw "you're on the latest version" with the button grayed out, even after v0.11.0 had shipped. Affected: every v0.10.2 install couldn't see any later release, including new features and important fixes.

  Temporary workaround (no longer needed as of v0.11.1): delete the `.git/` folder in the install directory and restart Studio. On startup it automatically reinitializes the repository and pulls down every version tag, after which Check for updates shows the new releases and you can upgrade through the webui as usual.
- **A preflight check wrongly blocked v0.10.2 users from confirming an update, claiming "the target version predates self-update" (#198)**
  Even when a v0.10.2 user could see v0.11.0 (for example via the workaround above), clicking "Confirm update → v0.11.0" hit a preflight check that popped a red X — "the target version predates the webui self-update feature — after switching you can only upgrade via CLI / shell (the webui can't rescue you)" — and disabled the confirm button, leaving no way forward.

  The misjudgment came from v0.11.0 moving server-side files: the v0.10.2 client looked at the old path and couldn't find the self-update detection flag. The already-released v0.10.2 client can't be hot-fixed, so this release keeps compatibility files in the repository so the old client's check passes. Once you've upgraded, the next upgrade follows the normal path and won't hit this again.
